CF1798D - Shocking Arrangment

发表于 2024-02-28 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1798

问题描述

You are given an array a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ consisting of integers such that a1+a2+…+an=0a_1 + a_2 + \ldots + a_n = 0a1​+a2​+…+an​=0.

You have to rearrange the elements of the array aaa so that the following condition is satisfied:

\max\limits_{1 \le l \le r \le n} \lvert a_l + a_{l+1} + \ldots + a_r \rvert &lt; \max(a_1, a_2, \ldots, a_n) - \min(a_1, a_2, \ldots, a_n),

where ∣x∣|x|∣x∣ denotes the absolute value of xxx.

More formally, determine if there exists a permutation p1,p2,…,pnp_1, p_2, \ldots, p_np1​,p2​,…,pn​ that for the array ap1,ap2,…,apna_{p_1}, a_{p_2}, \ldots, a_{p_n}ap1​​,ap2​​,…,apn​​, the condition above is satisfied, and find the corresponding array.

Recall that the array p1,p2,…,pnp_1, p_2, \ldots, p_np1​,p2​,…,pn​ is called a permutation if for each integer xxx from 111 to nnn there is exactly one iii from 111 to nnn such that pi=xp_i = xpi​=x.

Input

Each test contains multiple test cases. The first line contains the number of test cases ttt (1≤t≤50 0001 \le t \le 50\,0001≤t≤50000). The description of the test cases follows.

The first line of each test case contains a single integer nnn (1≤n≤300 0001 \le n \le 300\,0001≤n≤300000) — the length of the array aaa.

The second line of each test case contains nnn integers a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ (−109≤ai≤109-10^9 \le a_i \le 10^9−109≤ai​≤109) — elements of the array aaa. It is guaranteed that the sum of the array aaa is zero, in other words: a1+a2+…+an=0a_1 + a_2 + \ldots + a_n = 0a1​+a2​+…+an​=0.

It is guaranteed that the sum of nnn over all test cases does not exceed 300 000300\,000300000.

Output

For each test case, if it is impossible to rearrange the elements of the array aaa in the required way, print “No” in a single line.

If possible, print “Yes” in the first line, and then in a separate line nnn numbers — elements a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ rearranged in a valid order (ap1,ap2,…,apna_{p_1}, a_{p_2}, \ldots, a_{p_n}ap1​​,ap2​​,…,apn​​).

If there are several possible answers, you can output any of them.

Solution

这是一道构造的题目，注意到题目有限定条件：∑ai=0\sum a_i=0∑ai​=0。

首先考虑有正有负的情况。此时，我们从左向右构造新数组，同时维护数组的前缀和 sss。

  • 如果 s≥0s\ge 0s≥0，那么就添加一个负数到末尾。
  • 如果 s<0s\lt 0s<0，那么就添加一个非负数到末尾。

因为保证了数列之和为 000，所以当 s≥0s\ge 0s≥0 时，你总能找到还没添加的非正数，同理第二种情况你也总能找到未添加的正数。

接下来，我们证明这个策略的正确性。

【结论】：min⁡ai≤s≤max⁡ai\min a_i\le s\le \max a_iminai​≤s≤maxai​

我们使用数学归纳法。当我们添加第一个数的时候，此时的 sss 就是某一个数 aia_iai​，这个结论显然成立。

随后，我们添加数。因为我们添加的数 aka_kak​ 和 sss 总是符号相反，所以

  • s≥0s\ge 0s≥0 时，那么如果要使结论不成立， ak+s<min⁡aia_k+s\lt \min a_iak​+s<minai​，所以 ak<min⁡aia_k\lt \min a_iak​<minai​，这与 min⁡ai\min a_iminai​ 的定义矛盾！
  • s<0s\lt 0s<0 时，同理。

所以，这样的策略下，这个结论总是成立的。

我们接着再证明在这样的策略下，对于任意区间，题目所给的不等式一定成立。因为任意区间可以写成两个前缀和的差，∑i=lrai=Sr−Sl−1\sum_{i=l}^{r} a_i=S_r-S_{l-1}∑i=lr​ai​=Sr​−Sl−1​，所以 ∣Sr−Sl−1∣≤max⁡ai−min⁡ai|S_r-S_{l-1}|\le\max a_i-\min a_i∣Sr​−Sl−1​∣≤maxai​−minai​（因为两个前缀和不可能同时取到 min⁡\minmin 和 max⁡\maxmax）

特殊情况是：全部为 000 是无解的。这里特判一下即可。
