CF1572A - Book

发表于 2024-02-28 更新于 2024-03-18 分类于 ACM ， Codeforces ， 1572

问题描述

You are given a book with nnn chapters.

Each chapter has a specified list of other chapters that need to be understood in order to understand this chapter. To understand a chapter, you must read it after you understand every chapter on its required list.

Currently you don’t understand any of the chapters. You are going to read the book from the beginning till the end repeatedly until you understand the whole book. Note that if you read a chapter at a moment when you don’t understand some of the required chapters, you don’t understand this chapter.

Determine how many times you will read the book to understand every chapter, or determine that you will never understand every chapter no matter how many times you read the book.

Input

Each test contains multiple test cases. The first line contains the number of test cases ttt (1≤t≤2⋅1041 \le t \le 2\cdot10^41≤t≤2⋅104).

The first line of each test case contains a single integer nnn (1≤n≤2⋅1051 \le n \le 2\cdot10^51≤n≤2⋅105) — number of chapters.

Then nnn lines follow. The iii-th line begins with an integer kik_iki​ (0≤ki≤n−10 \le k_i \le n-10≤ki​≤n−1) — number of chapters required to understand the iii-th chapter. Then kik_iki​ integers ai,1,ai,2,…,ai,kia_{i,1}, a_{i,2}, \dots, a_{i, k_i}ai,1​,ai,2​,…,ai,ki​​ (1≤ai,j≤n,ai,j≠i,ai,j≠ai,l1 \le a_{i, j} \le n, a_{i, j} \ne i, a_{i, j} \ne a_{i, l}1≤ai,j​≤n,ai,j​=i,ai,j​=ai,l​ for j≠lj \ne lj=l) follow — the chapters required to understand the iii-th chapter.

It is guaranteed that the sum of nnn and sum of kik_iki​ over all testcases do not exceed 2⋅1052\cdot10^52⋅105.

Output

For each test case, if the entire book can be understood, print how many times you will read it, otherwise print −1-1−1.

Solution

我们首先把章节之间的关系抽象成一条有向边：如果 a→ba\to ba→b，那么就说明 aaa 是 bbb 的前置章节。只有所有前置章节被读完才能读这一章，我们可以联想到拓扑排序。

不过和一般的拓扑排序不同的是，我们每次读，只能顺序的读，如果先读了 uuu，那么想要再次读 u−1u-1u−1 只能放到下一轮（也就是总的阅读次数 +1+1+1）。所以我们可以把阅读的前后顺序也抽象一下，抽象成有向边的边权，如果 wa→b=1w{a\to b}=1wa→b=1，就说明 aaa 虽然是 bbb 的前置，但是由于我们先阅读 aaa，然后才能去阅读 bbb，那么阅读 bbb 只能放到下一轮，也就是总的阅读次数 +1+1+1；如果 wa→b=0w{a\to b}=0wa→b=0，就说明可以同一个轮次里先阅读 aaa 再阅读 bbb。

求解次数就可以转换成一个 DAG\text{DAG}DAG 上的 DP\text{DP}DP 问题，这里，DP\text{DP}DP 策略的选择应该是设置成 dpi=0dp_i=0dpi​=0 然后取 max⁡\maxmax 更新，而不是设置成 INF\text{INF}INF 然后取 min⁡\minmin 更新。原因在于，考虑 aaa 有前置 b,cb,cb,c，并且有 b<a,c>ab<a,c>ab<a,c>a，那么如果取 min⁡\minmin 更新，那么 dpa=min⁡(dpb+wb→a,dpc+wc→a)dp_a=\min(dp_b+w{b\to a},dp_c+w{c\to a})dpa​=min(dpb​+wb→a,dpc​+wc→a)，然而这个 DP\text{DP}DP 表达式的实际含义是只要 b,cb,cb,c 其中的一个被阅读了，我就可以读 aaa，这与题意是不符的；取 max⁡\maxmax 的实际含义才是只有全部读完前置才能读 aaa。

当然，有可能有无解的情况。这种情况下，就说明 DAG\text{DAG}DAG 上有环。我们可以统计拓扑处理过的点的个数 cnt，如果 cnt 不等于总的点数，就说明有环，永远不可能读完前置。
