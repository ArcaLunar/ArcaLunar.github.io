CF1933E - Optimal Training

发表于 2024-02-27 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1933

原题链接

问题描述

Isaac 开始他的训练。有 $n$ 条跑道可用，第 $i$ 条跑道 （$1 \le i \le n$）由 $a_i$ 个等长的小节组成。

给定整数 $u$（$1 \le u \le 10^9$），完成每个小节可以增加 Isaac 的能力一定值，描述如下：

  • 完成第 $1$ 个小节会使 Isaac 的表现提高 $u$。
  • 完成第 $2$ 个小节会使 Isaac 的表现提高 $u-1$。
  • 完成第 $3$ 个小节会使 Isaac 的表现提高 $u-2$。
  • $\ldots$
  • 完成第 $k$ 个小节（$k \ge 1$）会使 Isaac 的表现提高 $u+1-k$。（值 $u+1-k$ 可以是负数，这意味着完成额外的小节会降低 Isaac 的表现。）

你还会得到一个整数 $l$。你必须选择一个整数 $r$，使得 $l \le r \le n$，Isaac 将完成每个跑道的每个小节 $l, l + 1, \dots, r$（即，总共 $\sum_{i=l}^r a_i = a_l + a_{l+1} + \ldots + a_r$ 个小节）。

回答以下问题：你可以选择的最佳 $r$ 是多少，使得 Isaac 的表现提升最大？

如果有多个 $r$ 可以使 Isaac 的表现提升最大，输出最小的 $r$。

为增加难度，你需要回答 $q$ 个不同的 $l$ 和 $u$ 值的问题。

Input

The first line of input contains a single integer $t$ ($1 \le t \le 10^4$) — the number of test cases.

The descriptions of the test cases follow.

The first line contains a single integer $n$ ($1 \le n \le 10^5$).

The second line contains $n$ integers $a_1, a_2, \ldots, a_n$ ($1 \le a_i \le 10^4$).

The third line contains a single integer $q$ ($1 \le q \le 10^5$).

The next $q$ lines each contain two integers $l$ and $u$ ($1 \le l \le n, 1 \le u \le 10^9$) — the descriptions to each query.

The sum of $n$ over all test cases does not exceed $2 \cdot 10^5$. The sum of $q$ over all test cases does not exceed $2 \cdot 10^5$.

Output

For each test case, output $q$ integers: the $i$-th integer contains the optimal $r$ for the $i$-th query. If there are multiple solutions, output the smallest one.

Solution

我们翻译一下题目的意思，其实就是在说：对于一个给定的 $a$ 数列、一个区间的左端点 $l$ 和一个 $u$，希望 $\sum_{i=l}^{r} a_i$ 尽可能接近 $u$。我们先想一下，如果区间的总和是 $k$，那么这一段区间能获得的能力值是 $S=u+\ldots+(u+1-k)=\frac{k\times(2u+1-k)}{2}$。我们希望 $S$ 尽可能接近 $u$，因为如果等于 $u+1$ 的话，$u+1-(u+1)=0$，他对能力值提升的贡献是 $0$，就可以扔掉这一段。

所以，我们需要快速处理出区间的和这一信息，这一部分可以使用前缀和实现。

接着，我们需要在前缀和上进行二分来查找总和最接近 $u$ 的区间。我们可以将 pre[x] - pre[l-1] > u 作为二分的分界线。但是，我们仍需要对这个分界点做一下讨论：因为可能分界点左右的区间和是 $u-1$ 和 $u+5$，而前者区间能获得的能力值大于后者，所以，需要额外判断一下。
