Module Structure

IO 模块通过 signal lines (e.g. system bus lines) 与计算机的其他部分连接。

传输到其他地方的 Data 和从其他地方接受的 Data 都 buffer 在 registers 中，也有一些 status register 提供状态信息。

模块通过 control lines 与 processor 通信，processor 可以通过 control lines 向 IO 模块发送指令。

Concepts of I/O

Memory-Mapped I/O 与 I/O-Ports

一些概念：

Register Usage
Status Register 反应设备的状态（正在读写，准备就绪与否，纸张已打印……）
Control Register 控制设备的操作（读/写，打印机纸张大小……）
Data Register 要读写的 data
/ CPU 通过读写 I/O 模块的 registers 来控制 I/O 模块的操作。

  • Port-Mapped I/O

    Registers 可以成为专用的 (dedicated) I/O port（如果 CPU 提供 direct I/O 操作的指令）

  • Memory-Mapped I/O

    Registers 放在 Memory Map 中，I/O Operation 需要以读写 memory address 的方式进行。

I/O Techniques

Overview

  • Programmed I/O

    Data 在 Processor 和 I/O Module 之间传输。Processor 拥有对 I/O Module 的直接控制。传输指令时，Processor 必须等 I/O Module 完成指令才能继续操作。（Drawback: 浪费处理器资源）

  • Interrupt-Driven I/O

    Processor 向 I/O Module 发送完指令后继续其他指令，当 I/O Module 完成时，Processor 的当前指令被中断（interrupt）

  • Direct Memory Access (DMA)

    Processor 负责从 Main memory 提取 data 和把 data 存放到 main memory 中，I/O Module 直接从 main memory 里读取/写入数据，无需通过 Processor 中继。

Programmed I/O

Control and Status Register (CSR) is used to control the operation of the device and report status.

Interrupt-Driven I/O

When I/O finishes, I/O module interrupts the CPU. The CPU will then suspend current program, execute the remaining I/O operations(as in Programmed I/O), then return to the suspended program.

硬件流程和软件流程：

Direct Memory Access (DMA)

DMA 引入新的 module，模拟 processor 和 从 processor 手里接过 system bus 的控制权，用来通过 system bus 向/从 memory 中传输数据。因此，DMA 必须在 processor 不工作时占用 system bus，或者干脆命令 processor 把当前进程 suspend，称为 cycle stealing。

当 processor 需要读写数据时，它向 DMA 发送一个命令，包含如下信息：

信息 说明
R/W Control Line 要读写时，通过 DMA 和处理器之间的 control line 来发送请求
Address of I/O device involved 通过 data line 传输
Start location of memory 内存地址的开头。通过 data line 传输，寄存在 DMA 的 Address Register 里
Number of Words 读写多少数据。data line 传输，保存在 Data Count Register 里

然后 processor 继续其他工作（它把 I/O operation 授权给 DMA），DMA 直接操作 memory（不经过 processor）。传输结束时，DMA 向 processor 发送 interrupt 信号。

【何时 processor 会 suspend】只在需要用到 system bus 的时候，pauses for one bus cycle，然后 DMA 传输 one word 并返回控制权。

DMA 的不同设计

DMA 和 I/O 模块可以合并起来：DMA 既可以作为 I/O module 的一部分，也可以是 DMA 作为独立的部分控制多个 I/O modules。更进一步地，DMA 可以通过 I/O bus 控制多个 I/O modules。在以上三种方案里，都只有 DMA 通过 system bus 和 memory 通信。

Chapter 12. Operating System Support

The OS is a software that controls the execution of programs on a processor and manages the processor’s resources.

  1. (Application Programs) The end user views a computer system in terms of an application , not concerned about the computer’s architecture.
  2. (AP & Utilities) Application can be expressed in a programming language , then as a set of processor instructions. To ease this task, a set of system programs is provided. Some of these programs are referred to as utilities.
  3. (Utilities) These implement frequently used functions that assist in program creation, the management of files, and the control of I/O devices.
  4. (OS) The OS masks the details of the hardware from the programmer and provides the programmer with a convenient interface for using the system. It acts as mediator, making it easier for the programmer and for application programs to access and use those facilities and services.

OS Services

  • Program creation

    The OS provides a variety of facilities and services, such as editors and debuggers, to assist the programmer in creating programs. Typically, these services are in the form of utility programs that are not actually part of the OS but are accessible through the OS.

  • Program execution

    A number of steps need to be performed to execute a program. Instructions and data must be loaded into main memory, I/O devices and files must be initialized, and other resources must be prepared. The OS handles all of this for the user.

  • Access to I/O devices

    Each I/O device requires its own specific set of instructions or control signals for operation. The OS provides a uniform I/O interface to the programs, and takes care of the details so that the programmer can think in terms of simple reads and writes. Also handle sharing of the devices among users to prevent users from holding the device indefinitely

  • Controlled access to files

    How files and directory are created, modified, hidden from the user. Also file protections, file sharing is managed by the OS.

  • System access

    In the case of a shared or public system, the OS controls access to the system as a whole and to specific system resources. The access function must provide protection of resources and data from unauthorized users and must resolve conflicts for resource contention.

  • Error detection and response

    Report error to programs, and their remedial action

  • Accounting

    Collect usage statistics for various resources and monitor performance parameters such as response time, page fault rate

Protection Scheme

  • CPU executes in different modes to facilitate protection.
    • User Mode
    • Supervisor Mode (Kernel Mode)
  • Some resources (such as I/O devices) or actions (some special instructions) can only be run in kernel mode, so that users will not have access to them.
  • OS is supposed to be well-tested, while bugs exist in user programs. If user programs are allowed to handle system resources, they may hang the system, or hold the resources indefinitely, if their program is buggy.
  • Only the OS (which execute in kernel mode), will have the rights .

OS Call Mechanism

  • System has to change from user mode to kernel mode to run/use the protected resources.
  • Users cannot change themselves to kernel mode .
  • The OS functions are usually accessed via special entry points called system calls.
  • The OS will change to kernel mode, and execute with higher privileges than user program.

Multitasking and Time-sharing System

  • CPU idles (闲置) when I/O is performed.

  • To fully utilize CPU resource, CPU is shared among processes (multiple tasks). 在 tasks 之间快速切换

    Each process is allocated a time slice of 0.1 sec, say.

    When process uses up its time slice, it will stop execution, and let the next process to be executed

  • When a process cannot proceed (e. g., perform I/O), it will also stop execution

Process Scheduling

  • The system maintain a queue of processes.
  • The order of the processes in the queue is controlled by many factors.
    • Process priority
    • How long it has been waiting
    • Whether it has used up its time slices
    • System loading
  • The first process in the process queue is picked for execution.

Memory Management

Virtual Memory

Physical Address VS. Logical Address
Terms Explanation
Physical Address low-level, hardware-specific and used by the hardware directly
Logical Address higher-level, software-friendly address that requires mapping or translation into a physical address before being used by the hardware

Virtual memory uses both hardware and software to enable a computer to compensate for physical memory shortages, temporarily transferring data from random access memory (RAM = physical memory) to disk storage. Mapping chunks of memory to disk files enables a computer to treat secondary memory as though it were main memory.

Memory Management Unit (MMU)

Similar to Cache memory, it is based on the principle of locality, and we use address mapping to map logical addresses (of programs) to physical addresses (of memory).

Paging

Memory is partitioned into equal fixed-size chunks that are relatively small.

Processes are also divided into small fixed-size chunks of same size.

The chunks of a program (called pages) could be assigned to available chunks of memory (called frames).

Memory 不足的解决办法：Logical Address + Page Table

如果 memory 空间不足了，可以利用 logical address。对于每个 process，OS 维护一个 page table，保存 frame location for each page。在程序本身内部，每一个 logical address 包含 page number 和 page 内的相对地址 (relative address within the page). With paging, the logical-to-physical address translation is still done by processor hardware. Presented with a logical address (page number, relative address), the processor uses the page table to produce a physical address (frame number, relative address).

Terms Explanation

P Protection Information. (more than 1 bit). indicate protection mode (read/write, kernel/user mode)
D Dirty bit. Whether this page has changed (to prevent unnecessarily write back when the page has not been changed, since writing back is expensive.)
Demand Paging (Similar to Cache)

When program starts, nothing is in memory. We only bring in pages on demand (when the page is needed).

  • Advantage

    fast response, because we do not need to bring in the entire program for it to start.

  • Disadvantage

    a lot of page faults (similar to cache miss), until a stable working set of the program has been brought into the memory. Use pre-paging to bring in a number of pages at the beginning to minimize this.

Address Translation

When the program access a memory location via the CPU address bus:

  1. extract the logical page number (the most significant n-bits)
  2. find the corresponding Page Table Entry and see if the page is in physical memory
    1. If YES, then get the physical frame number and append the offset within the page to it.
    2. If NO, then generate a page fault.
Address Mapping

If the page is not in main memory, then a page fault is generated. The OS will take over, fetch the required page from hard disk, put it in the memory (possibly with a replacement) and restart the program.

Since hard disk I/O is very slow, the process involved will usually be suspended and the CPU will switch to another process.

Page Fault Handling
  1. Find the page from hard disk
  2. Find a free page in the physical memory (or find one to replace)
  3. If the replaced page is dirty (i.e. has been changed), write back to hard disk.
  4. Make PTE of replaced page invalid.
  5. Write required page to that physical page.
  6. Modify the PTE of required page to contain correct info.
Page Replacement
  • First In First Out (FIFO)
  • Least Recently Used (LRU)
Write Strategy
  • Write back strategy is always used.
  • Write through to hard disk is too expensive.
Translation Lookaside Buffer (TLB)

Virtual Memory 也会导致两次读取 Physical Memory：一次 fetch PTE，一次 fetch data。为了减少读取次数，把 valid PTE cache 在 TLB 里。那么在 Address Translation 的时候，先在 TLB 里查 PTE；找不到的话，再去 Page Table 查找（此时已经发生 Page Fault），所以接着写入 Memory，最后把对应的 PTE 放进 TLB 并更新。

Combining Cache and Virtual Memory

对于程序而言，我们先用 logical address 在 TLB/Page Table 里查找出 physical address（virtual memory，本章），然后再用 physical address 在 cache memory 里查找数据（cache，第 7 章）。

Instruction Pipeline

如何提升 throughput (吞吐量)（提升效率）？划分为六个阶段，pipeline。

多个 instruction 同时执行不同阶段

  • Note that not all instruction require all stages
  • The resources (hardware) required by each stages should not overlap with each other, otherwise duplicate resources are needed
    • 多个 internal bus，而不是一个（为了同时传输数据）
    • dedicated incrementor for PC instead of using ALU（PC 的专用自增器，ALU 可能用来做其他计算工作）
    • Separate read port for data and instruction (FI vs FO)
    • May have separate instruction cache and data cache (split cache)
Pipeline Hazard

Situations that prevent the next instruction from entering the pipeline (for execution)（另一种解释：when the next instruction cannot execute on the following clock cycle.）

  1. Resource Hazard/Structural Hazard

    Resource Conflict.

    1. If only one multiplier unit available, and current and next instruction both require multiply, the next instruction must wait for previous multiply to finish.
    2. 同时读取 instruction 和 data

    Solutions:

    1. Add more resources (dedicated incrementor for PC)
    2. separate data/instruction cache (split cache), so that FI and FO stage will use different hardware.

  2. Data Hazard

    instruction waiting for the result of previous instructions. (data dependency)

    1. Rearrange instruction
    2. You may need to insert two or more instruction in between, depending on how the pipeline is arranged and optimized.
    3. forwarding the result of ADD from the output of the ALU to the input of ALU when SUB is executed. (This is called data forwarding)

    • Types of Data Hazard

      • Read after Write (RAW) Hazard

        I1I_1I1​ writes to a register and I2I_2I2​ reads the same register. Hazard occurs if the read is before the write

      • Write after Read (WAR) Hazard

        I1I_1I1​ reads a register and I2I_2I2​ writes the same register. Hazard occurs if the write is before the read.

      • Write after Write (WAW) Hazard

        I1I_1I1​ and I2I_2I2​ both writes to the same register, and I2I_2I2​ writes before I1I_1I1​.

      Only RAW hazard occurs in a pipeline. (Each stage of the later instruction will only be dispatched after the same stage of the previous instruction finishes.) The others (WAR and WAW) occurs in parallel systems.

  3. Control Hazard/Branch Hazard

    arises from branches or other instructions that changes the PC (e.g. CALL/JSR, jump subroutine). We don’t know where to continue until the branch instruction finishes.

    • Branch Prediction：提前（预测并）获取 branch 内的指令，当条件被执行时可以立刻执行指令（不用再 fetch）

      ​ For simple branch prediction, we will change our next prediction when there is a wrong prediction. （Drawback：循环的最后一层

    • Dynamic Branch Prediction：连续错误预测两次才改变 prediction。

如何提升性能？

  • Exploit more effective instruction pipelining. This will lead to a smaller value of CPI. (The throughput of instruction execution is higher with pipelining).
  • use multiple instruction execution unit.
  • large register file — reduce memory access, hence increasing speed
  • simplified instruction set — reduces or eliminates the need for microprograms, which is slower than hardwired implementation

Characteristics of Modern Processor - RISC

  • All instructions are register-register type except LOAD and STORE (which access memory)
  • Fixed length and simple, fixed format instructions (格式中多余的部分可以直接丢弃)
  • Relatively few operations and addressing modes — Simple CPU, simpler CPU implementation, faster clock rate. Also, the pipeline can be designed more efficiently, if the are less cases to consider.
  • Use of hardwired rather than microprogrammed control (多用硬件控制)
  • Use of instruction pipelining and extensive software and hardware techniques to eliminate the effect of pipeline hazards (减少 hazards)
