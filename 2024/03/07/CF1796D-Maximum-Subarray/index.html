CF1796D - Maximum Subarray

发表于 2024-03-07 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1796

题面描述

You are given an array a1,a2,…,ana_1, a_2, \dots, a_na1​,a2​,…,an​, consisting of nnn integers. You are also given two integers kkk and xxx.

You have to perform the following operation exactly once: add xxx to the elements on exactly kkk distinct positions, and subtract xxx from all the others.

For example, if a=[2,−1,2,3]a = [2, -1, 2, 3]a=[2,−1,2,3], k=1k = 1k=1, x=2x = 2x=2, and we have picked the first element, then after the operation the array a=[4,−3,0,1]a = [4, -3, 0, 1]a=[4,−3,0,1].

Let f(a)f(a)f(a) be the maximum possible sum of a subarray of aaa. The subarray of aaa is a contiguous part of the array aaa, i. e. the array ai,ai+1,…,aja_i, a_{i + 1}, \dots, a_jai​,ai+1​,…,aj​ for some 1≤i≤j≤n1 \le i \le j \le n1≤i≤j≤n. An empty subarray should also be considered, it has sum 000.

Let the array a′a'a′ be the array aaa after applying the aforementioned operation. Apply the operation in such a way that f(a′)f(a')f(a′) is the maximum possible, and print the maximum possible value of f(a′)f(a')f(a′).

Input

The first line contains a single integer ttt (1≤t≤1041 \le t \le 10^41≤t≤104) — the number of test cases.

The first line of each test case contains three integers nnn, kkk and xxx (1≤n≤2⋅1051 \le n \le 2 \cdot 10^51≤n≤2⋅105; 0≤k≤min⁡(20,n)0 \le k \le \min(20, n)0≤k≤min(20,n); −109≤x≤109-10^9 \le x \le 10^9−109≤x≤109).

The second line contains nnn integers a1,a2,…,ana_1, a_2, \dots, a_na1​,a2​,…,an​ (−109≤ai≤109-10^9 \le a_i \le 10^9−109≤ai​≤109).

The sum of nnn over all test cases doesn’t exceed 2⋅1052 \cdot 10^52⋅105.

Output

For each test case, print one integer — the maximum possible value of f(a′)f(a')f(a′).

解法一：线段树 O(Nlog⁡N)O(N\log N)O(NlogN)

我盟先做一下转换：kkk 个数 +x+x+x，其余 −x-x−x   ⟺  \iff⟺ 所有数 −x-x−x，kkk 个数 +2x+2x+2x。

我们思考一下最终选择的子数组里，有多少个数 +x+x+x，有多少个数 −x-x−x。根据贪心的性质，

  • 如果 x≥0x\ge 0x≥0，那么我肯定希望最后的子数组里有尽可能多的数 +x+x+x，但是能 +x+x+x 的数最多有 kkk 个，所以最优情况下，我选择 kkk 个连续的数 +x+x+x，那么最后的子数组也肯定会包含这 kkk 个数中的一部分，也就是说这 kkk 个数要尽量选满在最大区间内。
  • 如果 x<0x\lt 0x<0，那么我肯定希望最后的子数组里有尽可能少的数 +x+x+x，但是一定会有 kkk 个数被 +x+x+x，所以最优情况下，我选择 kkk 个连续的数 +x+x+x，那么最后的子数组肯定出现在其余 N−2kN-2kN−2k 个数中，也就是说这 kkk 个数要尽量选在两边，远离最大区间。

我们可以归纳一下这两种情况，先选 KKK 个数 +2x+2x+2x，查询区间的最大子区间和，然后将所选区间向右循环移动。注意，当 x<0x<0x<0 时，我们做一个转化：x=−xx=-xx=−x、k=N−kk=N-kk=N−k。

所以，问题转化成了单点修改、区间查询最大子段和的问题，我们可以用线段树维护。

解法二：动态规划 O(Nk)O(Nk)O(Nk)

我们也可以考虑用动态规划解决这个问题。我们定义 dpi,jdp_{i,j}dpi,j​ 表示前 iii 个数中，选择了 jjj 个数 +x+x+x、并且子数组以 aia_iai​ 结尾的最大子数组和。

  • 当我们选择 j=0j=0j=0 个数时，这个时候根据 jjj 的定义，aia_iai​ 也不 +x+x+x，不过在尝试拓展最大子数组的时候还是要尝试把之前的子数组拼在自己前面，所以 dpi,0=max⁡(ai−x,ai−x+dpi−1,0)dp_{i,0}=\max(a_i-x, a_i-x+dp_{i-1,0})dpi,0​=max(ai​−x,ai​−x+dpi−1,0​)，这个 dpi−1,0dp_{i-1,0}dpi−1,0​ 就是尝试拼在自己前面。
  • 当 j>0j>0j>0 时：
    • 如果当前元素 aia_iai​ +x+x+x，那么留给前 i−1i-1i−1 个数能 +x+x+x 的个数就只剩 j−1j-1j−1 了，所以 dpi,j=ai+x+max⁡(0,dpi−1,j−1)dp_{i,j}=a_i+x+\max(0,dp_{i-1,j-1})dpi,j​=ai​+x+max(0,dpi−1,j−1​)。
    • 如果当前元素 aia_iai​ −x-x−x，那么前面 i−1i-1i−1 个数需要有 jjj 个数 +x+x+x，这里有个隐藏条件：j≤i−1j\le i-1j≤i−1，此时 dpi,j=max⁡(dpi,j,ai−x+max⁡(0,dpi−1,j))dp_{i,j}=\max(dp_{i,j},a_i-x+\max(0,dp_{i-1,j}))dpi,j​=max(dpi,j​,ai​−x+max(0,dpi−1,j​))。

需要注意的一点是，我们枚举的 jjj 并不是一直 0∼K0\sim K0∼K，而是 max⁡(0,k−(N−i))∼min⁡(i,k)\max(0,k-(N-i))\sim \min(i,k)max(0,k−(N−i))∼min(i,k)。下界的意义是，前 iii 个数极端情况下（+x+x+x 最少），就是 +x+x+x 都发生在剩下的 N−iN-iN−i 个数上，所以前 iii 个最少有 k−(N−i)k-(N-i)k−(N−i) 个 +x+x+x，当然也要 ≥0\ge 0≥0；上界的意义是，前 iii 数最多 iii 个数 +x+x+x，而 +x+x+x 的次数也有限，最多为 kkk。
