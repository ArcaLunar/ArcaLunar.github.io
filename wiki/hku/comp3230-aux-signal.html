【COMP3230 补充知识】 系统信号处理

Signal Handling

Simple Signal Handler

signal() Function

  • signum: 要处理的 signal
  • sighandler_t
    • 可以是一个函数，签名是 void handler(int signum);
    • SIG_IGN: 忽略信号
    • SIG_DFL: 设为系统默认
  • 返回: 系统先前如何处理这个信号
  • 如果失败了的话，返回 SIG_ERR

signaction() Function

三步设置信号处理：

  1. 获取旧信号处理：sigaction(signum, NULL, &act)
  2. 修改信号处理器 act
  3. 设置信号处理器为新的：sigaction(signum, &act, NULL)

关于 struct sigaction

Member 说明
void (*sa_handler)(int); sa_handler 函数指针，指向处理信号的函数
void (*sa_sigaction)(int, siginfo_t *, void *); 也是处理信号的函数，但是可以从系统获取更多信息。
sigset_t sa_mask; 保存过滤出来的信号
int sa_flags; Specifies a set of flags which modify the behavior of the signal
Method 1

Blocking Signals

有的时候，如果多个信号在 sigaction 结构体被创建之前被程序接收的话，sigaction 就只能触发一次。因此我们需要机制来阻塞信号

用 mask 过滤 Signal

我们也可以选择用 mask 来过滤要处理的信号。

函数原型：sigprocmask(int how, const sigset_t *set, sigset_t *oldset);

  • 参数说明
    • how 有三种取值
      • SIG_BLOCK 把 input signal mask 加入当前的 signal mask
      • SIG_UNBLOCK 把 input signal mask 从当前的 signal mask 中去掉
      • SIG_SETMASK 直接设置当前的 signal mask 为 input signal mask
    • 注意 ： 所有的加入、去除都是在 sigset_t 类型的变量上操作的
  • 返回值
    • 0 表示设置成功
    • -1 表示设置失败
  • 配套函数
    • sigemptyset(sigset_t *set); 清空 set 保存的所有 signal mask
    • sigaddset(sigset_t *set, SIGNAL); 把 SIGNAL 信号加进 set 里

在下面的这个例子里，sigset_t blkset 定义了 sigset 变量，用来控制过滤 signal。

通过调用 signal(SIGINT, sigint_handler); 在程序的前 101010 分钟，代表程序终止的 SIGINT 信号被程序捕获后不会按系统默认那样自动退出，而是转而执行 sigint_handler(int signum) 函数

之后，通过 sigemptyset(&blkset); 先清空 signal mask，方便加入要过滤的 signal；然后通过 sigaddset(&blkset, SIGINT); 把 SIGINT 信号放入要过滤的信号集合里；最后调用 sigprocmask(SIG_BLOCK, &blkset, NULL); 阻塞所有的 SIGINT 信号

值得注意的是，在 Linux 内核里，如果有多个 SIGINT 信号，那么 sigaction() 会自动对后续的 SIGINT 作合并处理。

等待 101010 秒后（这 101010 秒里按 Ctrl + C 不会退出程序），sigprocmask(SIG_UNBLOCK, &blkset, NULL); 将 blkset 设置为 SIG_UNBLOCK，于是程序重新开始接收 SIGINT 信号并交由 sigint_handler() 处理。

随后 signal(SIGINT, SIG_DFL); 又重新把 SIGINT 的处理设为系统默认（即退出）。此时按下 Ctrl + C 即可退出程序
