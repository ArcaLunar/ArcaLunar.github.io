【COMP3230 补充知识】 POSIX Thread 编程

POSIX Thread Programming

参考文章

  • 为什么采用 POSIX Thread？
    • 是 Unix 系 OS 的常用标准
  • 不同系统有共同的 API，但是不同库的实现不一定一样

Pthreads in Linux

在 Linux 里，POSIX Threads 实现为 Standard Process (Light Weighted Process)。同一个 Application 里的 Threads 共享 Memory Address Space, open files, 全局变量……

并且，Linux 里用 clone() 创建线程，而不用 fork() 创建进程。这是为了做出区分，用 clone() 的话可以指定分配给 Threads 的资源。

Thread 相关的 API

创建 Thread

函数签名非常复杂。

  1. 第一个 *thread，保存 thread 的变量
  2. 第二个参数 *attr (attribute 的简写)，可以设置 thread 的属性，设置成 NULL 就是默认。
  3. thrfunc 是 C 语言的函数指针，指向一个函数。当 thread 被创建的时候就执行这个 thrfunc 所指的函数
  4. args single argument passed to the function. 如果有多个参数，需要打包成结构体。

Thread 主动返回

  • 结束 caller 线程
  • retval 线程的返回值
    • 在另一个线程里调用 pthread_join() 来获取这个线程的返回值。
    • 需要自己手动分配空间
  • 不会关闭已经打开的文件。必须在退出前手动关闭。（否则在线程结束后也会一直开着）

从外部终止 Thread

  • thread：要终止的线程

Thread 处理 Calcellation 信号

可以通过函数，改变默认处理信号的行为。当收到 cancellation 信号后，会在下一个 cancellation point 被终止

Cancellation point 可以理解为发生了 System Call 的地方。

  • changes the cancellation state for the calling thread to
    1. ignore cancellation requests – PTHREAD_CANCEL_DISABLE, or
    2. not ignored – PTHREAD_CANCEL_ENABLE (default)

changes the type of responses to cancellation requests to
• (1) cancel the calling thread as soon as the cancellation request is received (but some implementations do not guarantee this) – PTHREAD_CANCEL_ASYNCHRONOUS, or
• (2) keep the cancellation request pending until the next cancellation point – PTHREAD_CANCEL_DEFERRED (default)

Waiting for a Thread to Exit

pthread_join() 阻塞运行，直到 id 为 thread_id 的线程 terminate.

  • value_ptr 期待的线程返回值。不关心的话可以设置为 NULL

Pthread Lock 线程锁

要上锁的 data type 称为 Mutex

变量类型为 pthread_mutex_t，使用前必须初始化.

上锁： pthread_mutex_lock()
释放锁： pthread_mutex_unlock()

当一个线程在使用锁，OS 会自动 block 所有其他线程。

mutex 使用完毕后，pthread_mutex_destroy(&lock) 把它删掉。

PThread Condition variable

pthread_cond_t

pthread_cond_wait()

pthread_cond_signal()/pthread_cond_broadcast()

pthread_cond_destroy(&cond)

Pthread Semaphore

  • 声明为 sem_t.
  • Init: sem_init(&k, 0, value=x);.
  • sem_wait(&k): 访问 integer，减 111，若小于 000 则放入 k 的 queue 里。
    • 也被称为 P()
  • sem_post(&k): 加 111，若 queue 里有线程，唤醒其中一个。
    • 也被称为 V()

MutEx - Binary Semaphores

Linux 设计核心：Mutex 与 Semaphore
Semaphore 的一个例子

  • 当 k=0/1k=0/1k=0/1 时，Binary Semaphore 就是 Lock.
    • 只允许 111 个线程 at its critical section
    • 初始化为 111.
    • 占用的时候，k=0k=0k=0；完成之后，k=1k=1k=1
Semaphore 与 MutEx

比如说 P(&mutex)，执行这一条语句的条件是 kmutex>0k_{mutex}\gt 0kmutex​>0，而当 kmutex≤0k_{mutex}\le 0kmutex​≤0 时，就会被阻塞。

这一点行为特性很类似 MutEx.

与 MutEx 的区别

  1. MutEx 由同一个 process/thread 负责 acquire 和 release
  2. Semaphore 本质是处理生产者消费者问题，其假设是一个 process/thread 要么是生产者要么是消费者。
