Routing Algorithms

Routing Networks

Routing Algorithm 的任务是在浩瀚的 Router 网络里，找到一条 Good Path 连接 Sending Host 和 Receiving Host.

What is Path?

Sequence of routers that packets traverse from sending host to receiving host

How to define Good?

May vary. “Least cost”, “least congestion”, “fastest”…

Abstraction

把 Network 视为无向图，Routers 作为节点，Physical Link 作为无向边。

Link State

Link State 算法以 Dijkstra’s Algorithm 作为模板，由于需要知晓整张无向图的信息，因此，Link State 是 Global 的，在实现上只能是 Centralized 的。

算法本身比较简单，实际上就是 Dijkstra 单源最短路算法的流程。

以 Source Node 为起点计算完 Dijkstra 算法后，会形成一张 Forward Table 保存在 Source Node 的 Router 里。

Algorithm Complexity
可以做到 O(nlog⁡n)O(n\log n)O(nlogn) 的复杂度

Message Complexity
每个 Router 都要给 Centre 发送自己到其他 nnn 个 Router的 Link 信息，收集这个信息是 O(n)O(n)O(n) 的。因此总时间复杂度是 O(n2)O(n^2)O(n2) 的。

Oscillation

在 Dynamic Load Cost 的情况下，可能会产生 Route Oscillation 的问题。

Distance Vector

Distance Vector 以 Bellman-Ford 算法为模板设计

Dx(y)=min⁡v{cx,v+Dv(y)}D_x(y)=\min_v\{c_{x,v}+D_v(y)\} Dx​(y)=vmin​{cx,v​+Dv​(y)}

这个算法是 Distributed 的。

  1. Key Point：每个 Router 向 Neighbor 发送自己的 Distance Vector Estimate
  2. 接受到 DV 之后，更新自己的 DV：
    1. Dx(y)←min⁡v(cx,v+Dv(y)),∀y∈ND_x(y)\gets \min_v(c_{x,v}+D_v(y)),\forall y\in NDx​(y)←minv​(cx,v​+Dv​(y)),∀y∈N

  • Iterative
  • Asynchronous
  • Distributed
  • Self-stopping

Another View

我们也可以把 DV 算法的流程看成是 State Information Diffusion.

  • 最开始的时候，只有 uuu 知道 uuu router 的信息
  • 随着事件的推移，越来越多的 router 知道 uuu 的信息。

Link Cost Changes

  • Node 检测到 Local Link Cost 改变
  • 更新 Routing 信息，计算新的 DV
  • 如果 DV 更新了，那么发送
