BGP：ISP 之间的路由选择

ISP 之间的路由选择

About BGP

BGP 是 ISP 之间的路由选择算法。

为什么需要 BGP？
  1. 在一个规模很大的网络里，路由信息的存储、传输和计算的开销巨大。

    考虑一个很大的网络，包含千万个 Router。如果使用 DV 算法，那么每一个 Router 都需要等待所有其他的 Router 的 DV 信息，收敛很慢；如果使用 LS 算法，就需要传输千万个 LS 信息，以及其存储和计算，资源开销巨大。

  2. 管理问题

    管理员希望可以管理子网络、对外隐藏细节、同时互联。

层次路由

两个层次：

  1. Inter-AS（不同的子自治区域之间）【外网】
  2. Intra-AS（每一个子自治区域）【内网】

优点：

  1. 解决管理问题
    1. AS 内部可以运行不同的 Routing Algo
    2. 可以隐藏内部的细节
  2. 解决规模问题
    1. AS 内部的路由数量不会太多
    2. Scalable，每增加一个子网，对于其他 AS 来说只是多了一项 Table Entry

BGP

尽管 AS 内部的 Routing Algo 可以不同，但是 AS 之间的 Routing Algo 必须相同，也就是 BGP 协议。BGP 相当于时把互联网的不同 AS 粘起来。

BGP 分成两个部分：eBGP (exterior) 和 iBGP (interior)。

  • eBGP：将这个 AS 内部的路由信息发送到外网
  • iBGP：将外网的路由信息发送到 AS 内部的其他 Router（包含网关）

基于改进的 DV 算法，同时告诉到达另一个 Router 经过的路径，这样可以检查出环，从而避免无穷计数问题。

BGP Session

两个 BGP Router 之间通过 Semi-permanent TCP 链接，并发送信息。

  1. Open 新建 TCP 连接用于 BGP 信息传输。
  2. Update Advertise new path
  3. KeepAlive 告诉对面保持 TCP 连接；也用于 ACK，表示 TCP 连接成功
  4. Notification 告诉对面上一条 Msg 有问题；也用于关闭 TCP 连接。

BGP Advertisement

一条 BGP Advertisement 包含：Prefix + Attributes

Prefix 就是 Destination to be Advertised.

两个最重要的属性：

  1. AS-PATH 到达目标的路径上，经过的 Hops
    1. 检测可能的环路
  2. NEXT-HOP 对于一种子网前缀，告诉你可以跳到哪一个 Hop

BGP 与 Forward Table

Hot-Potato Routing

Forward Table Entry：是 Intra-AS Routing 和 Inter-AS Routing 运算的共同结果。Intra-AS 里取 Cost 最低的。
