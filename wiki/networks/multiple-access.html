Multiple Access Protocol

有两种 Link：一种是 Point-to-Point，另一种是 broadcast（特点是 传播媒介是共享的）

我们主要考察 Single Shared Broadcast Channel，因为是 Shared，所以很可能发生 Collision 的问题，Node 同时接收到多个信号。这是就需要 Protocol 来决定如何处理了

Multiple Access Protocol

  1. 是一种 Distributed 算法，来决定哪一个 Node 在何时可以 Transmit Data
  2. 并且只能使用这个 Channel

理想情况下，假设 Link 的传输速率是 RRR：

  1. 只有一个 Node 的时候，可以达到 RRR
  2. MMM 个 Node 的时候，每一个可以达到 RMR\over MMR​
  3. 完全的去中心化
    1. 无需辅助节点协调 Nodes
    2. 无需考虑同步异步的问题
  4. 简单

我们主要考察 333 个 Protocol：

  1. Channel Partition
  2. Random Access
  3. Taking Turns

Channel Partition

Channel Partition

  • 将 Channel 划分为不同的 Slots（可以按 Frequency，Time 等等）
  • 将每一个 Slot 分配给一个 Node 供其专用
TDMA: Time
FDMA: Frequency
  • 以 Time 作为划分 Slot 的依据。并且将时间段等分
  • 每个 Node 可以分配到一部分时间段来发送数据。
  • 空闲的时间段就空闲着。
  • 以 Frequency 作为划分 Slot 的依据。划分为不同 Frequency Bands
  • 每个 Node 分配给一个独特的 Frequency band
  • 没被分配的 Frequency Band 就空闲着

Random Access

  • 每个 Node 都是直接以 RRR 的速率发送 Data，没有事先 Coordinate。
  • 如果无事发生，那就 OK 了；否则发生了 Collision，我们就需要以一定的策略处理 Collision
Slotted ALOHA
Pure ALOHA
CSMA

假设：

  1. 所有要发送的 Frame 大小一样
  2. Time 被划分为等长的时间段
  3. Node 只在 Slot 开始的时候发送 Frame
  4. Node 之间是 Sync 的
  5. 如果检测到 Collision，那么所有 Node 都会知道。

如果 Node 检测到 Collision，那么就把他的 Frame 放到下一个 Time Slot 传输。

如果又失败了，在接下来的每一个 Time Slot 里，以 ppp 的概率重新发送 Frame 直到成功。（几何分布）

Pros:

  1. 单个的 Active Node 总是可以占满 Bandwidth
  2. 高度去中心化
  3. 简单

Cons:

  1. 冲突的话，很浪费 Slot，很多空闲
  2. 需要实现 Node 主动检测 Collision
  3. 解决 Sync 问题

就是去掉 Sync 的 Slotted ALOHA：收到 Frame 就立刻 transmit

CSMA 主要想法

Listen Before Transmit. 先检测是否会 Collide，再 Transmit。

但是 Collision 还是可能发生，因为检测也是有延迟的（Propagation Delay）

加强版本：CSMA/CD

  1. Ethernet 接收 Datagram，创建 Frame
  2. 检测 Channel 是否 Idle，是的话发送 Frame，否则等待直到 idle
  3. 如果 Frame 成功发送出去了且没有 Collision，结束
  4. 否则检测到其他 Transmission，立刻中止，并发送 Jam Signal
  5. 中止后，进入 Binary Backoff 阶段：
    1. 在第 mmm 次 Collision 后，随机选择 K∈[0,2m−1]K\in[0,2^m-1]K∈[0,2m−1]
    2. Ethernet 等待 512m512m512m bit times，返回 Step 2
  6. 重复，直到发送完毕

Taking Turns

Polling Method
Token Passing Method
  • 一个中心化的 Controller 向其他 Node 发送 Invitation 来发送 Frame

劣势：

  1. Latency
  2. Single Point Failure (Controller)
  3. Polling Overhead
  • 循环的将 Token 传递给下一个 Node。
  • 只有持有 Token 的可以发送 Frame，不需要发送 Frame 的直接将 Token 传给下一个。

劣势：

  1. Token Overhead
  2. Single Point Failure (Token)
  3. Latency
案例学习：Cable Access Network

【背景】

  1. 多个 Downstream FDM channels
    • single CMTS transmits into channels​
  2. 多个 Upstream channel
    • Multiple Access (Random Access)：多个设备使用 Random Access 争夺 upstream channel time slots；其他的用 TDM。
