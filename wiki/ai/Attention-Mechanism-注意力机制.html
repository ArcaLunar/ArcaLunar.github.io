Attention Mechanism

一言以蔽之，对于一个查询 qqq 想知道其意思，你有一个字典 Ki→ViK_i\to V_iKi​→Vi​，你希望找到字典里和 qqq 最相似的 KiK_iKi​（就像查字典一样），并且你认为意思的相似程度和 key 的相似程度一致，于是 qqq 的意思就是 ∑isim(q,Ki)⋅Vi\sum_i sim(q,K_i)\cdot V_i∑i​sim(q,Ki​)⋅Vi​

我们有一个字典

D=def{(k1,v1),…(km,vm)}\mathcal{D} \stackrel{\textrm{def}}{=} \{(\mathbf{k}_1, \mathbf{v}_1), \ldots (\mathbf{k}_m, \mathbf{v}_m)\}D=def{(k1​,v1​),…(km​,vm​)}

这里的字典其实就是

对于一个 Query qqq，定义其 attention over D\mathcal{D}D 为

Attention(q,D)=def∑i=1mα(q,ki)vi\textrm{Attention}(\mathbf{q}, \mathcal{D}) \stackrel{\textrm{def}}{=} \sum_{i=1}^m \alpha(\mathbf{q}, \mathbf{k}_i) \mathbf{v}_i Attention(q,D)=defi=1∑m​α(q,ki​)vi​

通常而言，我们还会对 α(⋅)\alpha(\cdot)α(⋅) 作 softmax，来确保 α(⋅)∈[0,1]\alpha(\cdot)\in[0,1]α(⋅)∈[0,1]

α(q,ki)=exp⁡(α(q,ki))∑jexp⁡(α(q,kj))\alpha(\textbf q,\textbf k_i)=\frac{\exp(\alpha(\textbf q,\textbf k_i))}{\sum_j \exp(\alpha(\textbf q,\textbf k_j))} α(q,ki​)=∑j​exp(α(q,kj​))exp(α(q,ki​))​

这个计算权重的式子，本质上是 vi\textbf{v}_ivi​ 的线性组合

一个一维向量 q\textbf qq 乘上矩阵 K\textbf KK 是一个单词查字典，一个二维矩阵 Q\textbf QQ 乘上矩阵 K\textbf KK 就是多个单词查字典（等于一句句子查字典）

Attention Scoring Functions, 更快速的计算 Pooling

上一个 Section 里的 Similarity 计算其实是很慢的，对于一个 q\textbf qq，它要和每一个 ki\textbf k_iki​ 计算一下距离，才能计算出 α(⋅)\alpha(\cdot)α(⋅).

而相比之下，现代的 Python 机器学习库对于矩阵的运算更加快速，搭配英伟达 CUDA 库的优化能够实现更加高效的运算。因此，实操中，我们更多用向量点积代替。反正目的都是为了计算和 key 的匹配程度，计算匹配程度的函数称为 Attention Scoring Function.

点积计算和 key 的匹配程度

Dot Product Attention

我们来考察高斯函数的指数部分

a(q,ki)=−12∥q−ki∥2=q⊤ki−12∥ki∥2−12∥q∥2.a(\mathbf{q}, \mathbf{k}_i) = -\frac{1}{2} \|\mathbf{q} - \mathbf{k}_i\|^2 = \mathbf{q}^\top \mathbf{k}_i -\frac{1}{2} \|\mathbf{k}_i\|^2 -\frac{1}{2} \|\mathbf{q}\|^2. a(q,ki​)=−21​∥q−ki​∥2=q⊤ki​−21​∥ki​∥2−21​∥q∥2.

这个点积的结果只与 q\textbf qq 有关系（后面两个都可以看作常数）。

为了避免梯度爆炸和梯度消失的问题，我们希望能够保证 a()a()a() 的大小。不妨设 q,ki∈Rd\textbf q,\textbf k_i\in\mathbb R^dq,ki​∈Rd，并且 qj,(ki)j\textbf q_j,(\textbf k_i)_jqj​,(ki​)j​ 互相独立、服从标准分布 N(0,1)\mathcal N(0,1)N(0,1)。 那么得到的结果

E[q⋅ki]=E[∑qj×(ki)j]=0Var(q⋅ki)=Var(∑)=dE[\textbf q\cdot \textbf k_i]=E[\sum \textbf q_j\times(\textbf k_i)_j]=0\\ Var(\textbf q\cdot\textbf k_i)=Var(\sum)=d E[q⋅ki​]=E[∑qj​×(ki​)j​]=0Var(q⋅ki​)=Var(∑)=d

为了让其方差仍然保持在 111，我们乘上一个 1d1\over\sqrt{d}d​1​，scoring function 就变成了

a(q,ki)=qTkida(\textbf q,\textbf k_i)=\frac{\textbf q^T\textbf k_i}{\sqrt d} a(q,ki​)=d​qTki​​

为了让每一个 a(⋅)a(\cdot)a(⋅) 的范围仍然落在 [0,1][0,1][0,1] 之间，我们像之前一样做一个 softmax()

α(q,ki)=softmax(a(q,ki))=exp⁡(q⊤ki/d)∑j=1exp⁡(q⊤kj/d)\alpha(\mathbf{q}, \mathbf{k}_i) = \mathrm{softmax}(a(\mathbf{q}, \mathbf{k}_i)) = \frac{\exp(\mathbf{q}^\top \mathbf{k}_i / \sqrt{d})}{\sum_{j=1} \exp(\mathbf{q}^\top \mathbf{k}_j / \sqrt{d})} α(q,ki​)=softmax(a(q,ki​))=∑j=1​exp(q⊤kj​/d​)exp(q⊤ki​/d​)​

Batch Matrix Multiplication

如果我们有多个 Di:Ki↦Vi\mathcal D_i:\textbf K_i\mapsto \textbf V_iDi​:Ki​↦Vi​ 和多个询问 Qi\textbf Q_iQi​ 想“多个单词同时查多个字典”，我们可以用 Batch Matrix Multiplitcation 来批量计算 QKT\textbf{QK}^TQKT，提升速度。

最后再对应乘上 Vi\textbf V_iVi​ 即可。

Q=[Q1,Q2,…,Qn]∈Rn×a×b,K=[K1,K2,…,Kn]∈Rn×b×c.\mathbf{Q} = [\mathbf{Q}_1, \mathbf{Q}_2, \ldots, \mathbf{Q}_n] \in \mathbb{R}^{n \times a \times b}, \\ \mathbf{K} = [\mathbf{K}_1, \mathbf{K}_2, \ldots, \mathbf{K}_n] \in \mathbb{R}^{n \times b \times c}. Q=[Q1​,Q2​,…,Qn​]∈Rn×a×b,K=[K1​,K2​,…,Kn​]∈Rn×b×c.

则 BMM 的定义就是

BMM(Q,K)=[Q1K1,Q2K2,…,QnKn]∈Rn×a×c.\textrm{BMM}(\mathbf{Q}, \mathbf{K}) = [\mathbf{Q}_1 \mathbf{K}_1, \mathbf{Q}_2 \mathbf{K}_2, \ldots, \mathbf{Q}_n \mathbf{K}_n] \in \mathbb{R}^{n \times a \times c}. BMM(Q,K)=[Q1​K1​,Q2​K2​,…,Qn​Kn​]∈Rn×a×c.

这里出现的三个系数 Wq∈Rh×q,Wk∈Rh×k,wv∈Rh\mathbf W_q\in\mathbb R^{h\times q},\mathbf W_k\in\mathbb R^{h\times k},\mathbf w_v\in\mathbb R^{h}Wq​∈Rh×q,Wk​∈Rh×k,wv​∈Rh 是为了解决他们维度不一致的问题，所以乘上系数保证维度一致。并且这三个系数是可学习的。

Multi-Head Attention

有的时候，我们希望我们的模型能够捕获同一句句子里的 dependency，也就是说，一句句子的开头 x0x_0x0​ 和结尾部分 xtx_txt​ 可能更加有关系，而单一 Attention 仅是均等地考虑 x0∼xt−1x_0\sim x_{t-1}x0​∼xt−1​

比如说，给定一个英文句子，考虑其中地一个单词 xix_ixi​，它可以既和冠词 xt−1x_{t-1}xt−1​ 有语法上的关系，又可以和 xt−2x_{t-2}xt−2​ 有语义上的关系。

语法关系和语义关系再用同一个 Attention 参数来处理就显得不太合适了。

多头注意力

所以有的时候，我们可以使用多头注意力 (Multi-Head Attention)。具体做法是，我们让 Queries, Keys, Values 经过 hhh 个不同的 Linear Projection 变换一下，然后让这 hhh 个变换后的 QKV 并行地经过多个 Attention 进行计算。然后，这 hhh 个 Attention 的输出再拼接起来，经过一个 Fully Connected Layer 转化为输出。

多头注意力机制

hi=f(Wi(q)q,Wi(k)k,Wi(v)v)∈Rpv,\mathbf{h}_i = f(\mathbf W_i^{(q)}\mathbf q, \mathbf W_i^{(k)}\mathbf k,\mathbf W_i^{(v)}\mathbf v) \in \mathbb R^{p_v}, hi​=f(Wi(q)​q,Wi(k)​k,Wi(v)​v)∈Rpv​,

其中的 Wi(q)∈Rpq×dq,Wi(k)∈Rpk×dk,Wi(v)∈Rpv×dv\mathbf W_i^{(q)}\in\mathbb R^{p_q\times d_q},\mathbf W_i^{(k)}\in\mathbb R^{p_k\times d_k},\mathbf W_i^{(v)}\in\mathbb R^{p_v\times d_v}Wi(q)​∈Rpq​×dq​,Wi(k)​∈Rpk​×dk​,Wi(v)​∈Rpv​×dv​ 都是可学习参数。f(⋅)f(\cdot)f(⋅) 是 Attention Pooling (Attention Scoring Function)。注意，在多头注意力里，不同注意力模块的参数是不共享的

Self Attention

Attention 机制关注的是不同序列之间的关系，而 Self-Attention 机制则关注同一序列内部的关系。这使得 Self-Attention 更加适合捕捉序列内部的依赖关系，并且能够并行计算，提高效率。Self Attention 的 QKV 都来自同一段序列。

Self Attention 的数学语言描述

假定我们有一串序列 xi∈Rd,i∈[1,n]\mathbf x_i\in\mathbb R^{d},i\in[1,n]xi​∈Rd,i∈[1,n]，Self Attention 输出一串序列 yi∈Rd,i∈[1,n]\mathbf y_i\in\mathbb R^d,i\in[1,n]yi​∈Rd,i∈[1,n]。

yi=f(xi,(x1,x1),…,(xn,xn))∈Rd\mathbf{y}_i = f(\mathbf{x}_i, (\mathbf{x}_1, \mathbf{x}_1), \ldots, (\mathbf{x}_n, \mathbf{x}_n)) \in \mathbb{R}^d yi​=f(xi​,(x1​,x1​),…,(xn​,xn​))∈Rd

Positional Encoding

如何把单词在句子中的位置这一信息也给编码进 Self-Attention 里呢？我们可以遵循 Vaswani 的方法，即用 sin⁡,cos⁡\sin,\cossin,cos 转化位置信息，以预先 fixed 或 learned 的方法嵌入位置信息。

假定输入张量 X∈Rn×d\mathbf{X} \in \mathbb{R}^{n \times d}X∈Rn×d 包含 nnn 个单词，每个单词被表示为一个 ddd 维向量。Position Encoding 输出一个 X+P\mathbf X+\mathbf PX+P，其中的 P∈Rn×d\mathbf P\in\mathbb R^{n\times d}P∈Rn×d，对于同一行 iii 上的相邻两个位置 2j,2j+12j,2j+12j,2j+1 是这样计算的

pi,2j=sin⁡(i100002j/d),pi,2j+1=cos⁡(i100002j/d).\begin{aligned} p_{i, 2j} &= \sin\left(\frac{i}{10000^{2j/d}}\right),\\p_{i, 2j+1} &= \cos\left(\frac{i}{10000^{2j/d}}\right).\end{aligned} pi,2j​pi,2j+1​​=sin(100002j/di​),=cos(100002j/di​).​
