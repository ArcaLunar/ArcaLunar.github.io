【数据结构】平衡树：板子的使用说明书

平衡树

FHQ Treap

板子

Template

使用说明

这个板子的元素顺序是从小到大。

1. 类型设定 (建立平衡树)

  1. size_type
    • Default: uint32_t
    • 树的大小、节点编号的数据类型
  2. priority_type
    • Default: uint32_t
    • 节点的随机权重值
  3. template <typename> typename NodeWrapper
    • 节点的类型
  4. MAX_NODE (size_type)
    • Default: 1 << 20
    • 最大的节点数量
例子：建立一棵平衡树

2. 模板支持的操作

2.1 插入

  1. .insert_by_rank(const key_type &key, size_type k, Modify &&modify) -> void

    根据位置插入元素

    • 参数说明：
      • key：要插入的值
      • k: 插入后，是第几个数？下标为 [0, size()]
      • modify: 匿名函数，可以对这个新插入的节点进行修改、初始化。
    • 时间复杂度：O(log⁡n)O(\log n)O(logn)

  2. .insert_by_key(const key_type &key, Modify &&modify) -> void

    根据值插入元素

    • Parameters
      • key: 要插入的 key
      • modify: 对节点进行修改。默认不修改
    • 若已有同名元素，插入在后面。
    • 时间复杂度：O(log⁡n)O(\log n)O(logn)
2.2 删除

  1. .erase_by_key(const key_type &key) -> bool

    根据值删除元素

    • Parameter
      • key: 要删除的 key.
    • Returns
      • bool: 是否删除成功，找不到元素则返回 false
    • 时间复杂度：O(log⁡n)O(\log n)O(logn)

  2. .erase_by_rank(size_type k) -> void

    根据位置删除元素

    • Parameter
      • k: 删除的下标。范围 [0, size()-1]
    • 时间复杂度：O(log⁡n)O(\log n)O(logn)
2.3 修改

  1. .modify_by_rank(const size_type k, Modify &&modify) -> void

    修改某一位置上的元素

    • Param
      • k: 位置。[0, size()-1]
      • modify: 匿名函数，对元素的修改
    • 时间复杂度：O(log⁡n)O(\log n)O(logn)

  2. .modify_by_key(const key_type &key, Modify &&modify) -> bool

    找到值，然后修改。没找到就返回 false

    • Param
      • key
      • modify
    • 时间复杂度：O(log⁡n)O(\log n)O(logn)
2.4 查询

  1. .kth(size_type k) -> node*

    返回第 k 个位置上的元素

    • Param
      • k: 位置（k∈[0,size()−1]k\in [0,size()-1]k∈[0,size()−1]）
    • Returns
      • node* 指向节点的指针
        【注意】不要用这个方法获取节点并修改，这样子做不会自动 pushup()/pushdown()，无法维护区间

  2. .rank(const key_type& key) -> size_type

    返回这个 key 对应的排名

    • Param
      • key: 值
    • Returns
      • size_type: 位置。[0,size()][0, size()][0,size()]

  3. .smaller_bound(const key_type& key) -> node*

    前驱节点，<key

  4. .lower_bound(const key_type& key) -> node*

    >= key

  5. .upper_bound(const key_type& key) -> node*

    > key

  6. .root() -> node*

  7. .size() -> size_type
