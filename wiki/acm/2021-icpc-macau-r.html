【构造】A. So I’ll Max Out My Constructive Algorithm Skills

我们随便取一种走法，如果这种走法的上山次数大于下山次数，那么就 reverse()，此时上山变下山，一定满足条件

【构造】D. Shortest Path Fast Algorithm

如何卡 SPFA.jpg

graph LR;
	A[1] -->|1e6| B[2];
	B -->|1| D[4];
	A -->|1| C[3];
	C -->|1e5| D;
	C -->|1| B;

这样的入队顺序是：1,3,4,2,41,3,4,2,41,3,4,2,4，444 出队 222 次

【模拟、线段树】E. Sandpile

根据题意模拟流程即可，用线段树辅助。如果可以成功模拟 nnn 轮，那么一定是 Recurrent 的；否则就直接输出结果就好。

线段树维护

  1. 区间最大值
  2. 最大值的位置
  3. 懒标记（区间加）

【贪心、最小生成树】K. Link Cut Tree

贪心的策略，注意到 ∑i=1N2i<2N+1\sum_{i=1}^N 2^i\lt 2^{N+1}∑i=1N​2i<2N+1，所以我们跑出一棵生成树之后，记录最小的、不在树上的边，然后直接在树/森林上 DFS 找路径。

不要用树链剖分，DFS 就够了（树剖会被卡空间）
