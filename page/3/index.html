CF1872G - Replace with Product

发表于 2024-03-07 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1872

题目描述

Given an array $a$ of $n$ positive integers. You need to perform the following operation exactly once:

  • Choose $2$ integers $l$ and $r$ ($1 \le l \le r \le n$) and replace the subarray $a[l \ldots r]$ with the single element: the product of all elements in the subarray $(a_l \cdot \ldots \cdot a_r)$.

For example, if an operation with parameters $l = 2, r = 4$ is applied to the array $[5, 4, 3, 2, 1]$, the array will turn into $[5, 24, 1]$.

Your task is to maximize the sum of the array after applying this operation. Find the optimal subarray to apply this operation.

Input

Each test consists of multiple test cases. The first line contains a single integer $t$ ($1 \le t \le 10^4$) — the number of test cases. This is followed by the description of the test cases.

The first line of each test case contains a single number $n$ ($1 \le n \le 2 \cdot 10^5$) — the length of the array $a$.

The second line of each test case contains $n$ integers $a_1, a_2, \ldots, a_n$ ($1 \le a_i \le 10^9$).

It is guaranteed that the sum of the values of $n$ for all test cases does not exceed $2 \cdot 10^5$.

Output

For each test case, output $2$ integers $l$ and $r$ ($1 \le l \le r \le n$) — the boundaries of the subarray to be replaced with the product.

If there are multiple solutions, output any of them.

题解

我们先想一个简单情况，只有两个数 $a,b$，那么只要 $ab>a+b$ 我就要把 $a,b$ 用乘积替换，而且，感性地来说，如果固定 $b$，那么我的 $a$ 越大，$ab$ 相比 $a+b$ 就越大，对于答案的贡献也越大，所以更应该把 $a$ 乘进子数组里。总结一下，$a$ 越大，对答案的作用也越大。

换而言之，$a$ “作用最小”的时候是 $a=2$，在最坏情况下，$\max a_i=2$，所以，我们需要判断当用乘积替换的时候，是不是一定能够保证大于其和。我们可以猜测，只要当 $\prod a_i\gt 2N$ 时，就一定满足 $\prod a_i\ge \sum a_i$。

当 $\prod a_i\le 2N$ 时，我们就可以暴力枚举了，因为此时数的个数很少，$O(N^2)$ 足够枚举。

CF1935C - Messenger in MAC

发表于 2024-03-06 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1935

题面描述

In the new messenger for the students of the Master’s Assistance Center, Keftemerum, an update is planned, in which developers want to optimize the set of messages shown to the user. There are a total of $n$ messages. Each message is characterized by two integers $a_i$ and $b_i$. The time spent reading the set of messages with numbers $p_1, p_2, \ldots, p_k$ ($1 \le p_i \le n$, all $p_i$ are distinct) is calculated by the formula:

$$
\Large \sum_{i=1}^{k} a_{p_i} + \sum_{i=1}^{k - 1} |b_{p_i} - b_{p_{i+1}}|
$$

Note that the time to read a set of messages consisting of one message with number $p_1$ is equal to $a_{p_1}$. Also, the time to read an empty set of messages is considered to be $0$.

The user can determine the time $l$ that he is willing to spend in the messenger. The messenger must inform the user of the maximum possible size of the set of messages, the reading time of which does not exceed $l$. Note that the maximum size of the set of messages can be equal to $0$.

The developers of the popular messenger failed to implement this function, so they asked you to solve this problem.

Input

Each test consists of multiple test cases. The first line contains a single integer $t$ ($1 \leq t \leq 5 \cdot 10^4$) — the number of test cases. The description of the test cases follows.

The first line of each test case contains two integers $n$ and $l$ ($1 \leq n \leq 2000$, $1 \leq l \leq 10^9$) — the number of messages and the time the user is willing to spend in the messenger.

The $i$-th of the next $n$ lines contains two integers $a_i$ and $b_i$ ($1 \le a_i, b_i \le 10^9$) — characteristics of the $i$-th message.

It is guaranteed that the sum of $n^2$ over all test cases does not exceed $4 \cdot 10^6$.

Output

For each test case, output a single integer — the maximum possible size of a set of messages, the reading time of which does not exceed $l$.

题解

我们的目标，是让 $S=\sum a_{p_i}+\sum |b_{p_{i+1}}-b_{p_i}|$ 在不超过 $L$ 的情况下，选择尽可能多的 $p$。借用贪心的思想，我们可以直观地想到，如果我的 $S$ 越小，就越有可能放更多的数进来。

由于题目并没有限制 $p_i$ 的顺序，而 $\sum |b_{p_{i+1}}-b_{p_i}|$ 因为带绝对值，所以我们可以认为的规定一个顺序，使得 $b_{p_1}\le b_{p_2}\le b_{p_3}\le \ldots \le b_{p_k}$。这启发我们按 $b_i$ 升序排序。

排完序之后呢？我们考虑贪心策略：只要总和还没超过 $L$ 我就一直往里面放。如果超过了 $L$，我就先把大数扔掉，直到合法为止。之所以先扔大的，是因为如果先扔小的，那很有可能要扔掉很多个小的，才能腾出足够空间放进去数，而先大的的话，很有可能只需要扔掉一两个就可以腾出足够空间了。所以，我们在这里需要一个支持删除最大值、插入值、查询最大值的数据结构，我们可以用 priority_queue 或者 multiset 实现。

CCPC2023秦皇岛 D - Yet Another Coffee

发表于 2024-03-03 更新于 2024-03-14 分类于 ACM ， CCPC ， 2023 ， 秦皇岛

题面描述

The girls of SYSU like drinking tea. But one day, they wanted a change and decided to try coffee in the next $n$ days.

Now Zayin, who always provides food and drinks for SYSU, will go to the shop to buy some coffee. She learns that the price of day $i$ is $a_i$. Meanwhile, she has $m$ coupons - the $i$ coupon can be used before day $r_i$ (inclusively) and can reduce the price of coffee on that day by $w_i$.

Notice that each coupon can be used only once and Zayin can use more than one coupon per day. The price can be a negative number after using the coupons.

Since the girls of SYSU still need to drink tea, Zayin decided to choose some days to buy coffee. Now she wants to know the minimum money she will spend (or the maximum money she can get) if she chooses exactly $k\ (1\le k \le n)$ days to buy coffee.

Input

The first line of the input contains a single integer $t\ (1\le t\le 10)$ — the number of test cases. The description of the test cases follows.

The first line contains two integers $n,m\ (1\le n, m\le 2\times 10^5)$ - the number of days and the number of coupons.

The second line contains $n$ integers $a_1,a_2,\cdots,a_n\ (1\le a_i \le 10^9)$ - $a_i$ denotes the price of coffee on day $i$.

Next $m$ lines, each line contains two integers $r_i,w_i\ (1\le r_i\le n,\ 1\le w_i\le 10^9)$, denoting that the $i$ th coupon can be used before day $r_i$ and can reduce the price by $w_i$.

Output

For each testcase, output $n$ integers - the $i$ th integer represents the minimum money Zayin will spend to buy coffee if she chooses exactly $i$ days to buy coffee.

Notice that the answer can be a negative integer.

题解

我们从 $k=1$ 时入手。如果只用一天买咖啡，那么贪心地考虑的话，一定是希望大量优惠券用在同一天买咖啡，这样打折的力度更大，那么我们就希望价格减去优惠的最终价格最小。所以，对于 $k=1$ 时，我们的策略就是，对于第 $i$ 张优惠券，我们在 $1\sim r_i$ 中找一个价格最低的时候，然后把优惠券用在这一天（记为 $m_i$）。

当 $k>1$ 时，优惠券用在其他时间（$t\in [1,r_i], t\ne m_i$）和用在 $m_i$ 是一样的，因为只要用了优惠券，那么价格就一定减少 $w_i$，这和优惠券在什么时候使用是无关的。

所以我们可以对优惠券按过期时间 $r_i$ 进行排序，从小到大维护价格的最低点，在这一点减去优惠力度，最后对最终价格排序并计算前缀和即可。

CF1937C - Binary Operation Wizard

发表于 2024-03-01 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1937

问题描述

This is an interactive problem.

There is a secret sequence $p_0, p_1, \ldots, p_{n-1}$, which is a permutation of ${0,1,\ldots,n-1}$.

You need to find any two indices $i$ and $j$ such that $p_i \oplus p_j$ is maximized, where $\oplus$ denotes the bitwise XOR operation.

To do this, you can ask queries. Each query has the following form: you pick arbitrary indices $a$, $b$, $c$, and $d$ ($0 \le a,b,c,d \le n$). Next, the jury calculates $x = (p_a \mid p_b)$ and $y = (p_c \mid p_d)$, where $|$ denotes the bitwise OR operation. Finally, you receive the result of comparison between $x$ and $y$. In other words, you are told if $x \lt y$, $x \gt y$, or $x = y$.

Please find any two indices $i$ and $j$ ($0 \le i,j \lt n$) such that $p_i \oplus p_j$ is maximum among all such pairs, using at most $3n$ queries. If there are multiple pairs of indices satisfying the condition, you may output any one of them.

Input

Each test contains multiple test cases. The first line contains the number of test cases $t$ ($1 \le t \le 10^3$). The description of the test cases follows.

Interaction

The first line of each test case contains one integer $n$ ($2 \le n \le 10^4$). At this moment, the permutation $p_0, p_1, \ldots, p_{n-1}$ is chosen. The interactor in this task is not adaptive. In other words, the sequence $p$ is fixed in every test case and does not change during the interaction.

To ask a query, you need to pick four indices $a$, $b$, $c$, and $d$ ($0 \le a,b,c,d \lt n$) and print the line of the following form:

  • “? a b c d”

After that, you receive:

  • “<” if $(p_a \mid p_b) \lt (p_c \mid p_d)$;
  • “=” if $(p_a \mid p_b) = (p_c \mid p_d)$;
  • “>” if $(p_a \mid p_b) \gt (p_c \mid p_d)$.

You can make at most $3n$ queries of this form.

Next, if your program has found a pair of indices $i$ and $j$ ($0 \le i, j \lt n$) such that $p_i \oplus p_j$ is maximized, print the line of the following form:

  • “! i j”

Note that this line is not considered a query and is not taken into account when counting the number of queries asked.

After this, proceed to the next test case.

If you make more than $3n$ queries during an interaction, your program must terminate immediately, and you will receive the Wrong Answer verdict. Otherwise, you can get an arbitrary verdict because your solution will continue to read from a closed stream.

After printing a query or the answer for a test case, do not forget to output the end of line and flush the output. Otherwise, you will get the verdict Idleness Limit Exceeded. To do this, use:

  • fflush(stdout) or cout.flush() in C++;
  • System.out.flush() in Java;
  • flush(output) in Pascal;
  • stdout.flush() in Python;
  • see the documentation for other languages.

It is guaranteed that the sum of $n$ over all test cases does not exceed $10^4$.

Hacks

To hack, follow the test format below.

The first line contains the number of test cases $t$ ($1 \le t \le 10^3$). The description of the test cases follows.

The first line of each test case contains one integer $n$ ($2 \le n \le 10^4$).

The second line of each test case contains $n$ integers $p_0,p_1,\ldots,p_{n-1}$, which represent a permutation of integers from $0$ to $n - 1$.

The sum of $n$ over all test cases should not exceed $10^4$.

Solution

我们可以猜测异或后的最大数一定是 $2^k-1$（即全 $1$）。那我们考虑哪些数异或后可以为全 $1$。比如说，我们枚举一下 $k$，可以发现总是可以通过 $N-1$ 与另一个数 $a$ 进行异或操作后得到 $2^k-1$。

所以，我们的策略如下。

  • 找到 $N-1$
  • 找到一个 $a$，使得 $(N-1) \oplus a=(111\ldots1)_2$

由于 $a\ |\ a=a$，所以，我们可以借助打擂台找最大值的思想，令 w=0，然后不断询问 w,w,i,i，如果 i|i 更大就更新 w，这样我们就获得了最大值 $N-1$ 的位置。

接着，我们去找 $a$。因为异或后全一，所以 $N-1$ 为 $0$ 的位置上，$a$ 一定为 $1$，因此 $(N-1)\mid a=(111\ldots 1)_2$。但是按位或操作不能检查其他位置上是否是 $0$，所以，我们先把满足 $(N-1)\mid x=(111\ldots 1)_2$ 的数存起来，然后找其中最小的数即可。

询问函数的实现

先找 $N-1$

再找 $x$

最后找 $x$ 中最小值，就是答案 $a$

CF1798D - Shocking Arrangment

发表于 2024-02-28 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1798

问题描述

You are given an array $a_1, a_2, \ldots, a_n$ consisting of integers such that $a_1 + a_2 + \ldots + a_n = 0$.

You have to rearrange the elements of the array $a$ so that the following condition is satisfied:

$$
\max\limits_{1 \le l \le r \le n} \lvert a_l + a_{l+1} + \ldots + a_r \rvert < \max(a_1, a_2, \ldots, a_n) - \min(a_1, a_2, \ldots, a_n),
$$
where $|x|$ denotes the absolute value of $x$.

More formally, determine if there exists a permutation $p_1, p_2, \ldots, p_n$ that for the array $a_{p_1}, a_{p_2}, \ldots, a_{p_n}$, the condition above is satisfied, and find the corresponding array.

Recall that the array $p_1, p_2, \ldots, p_n$ is called a permutation if for each integer $x$ from $1$ to $n$ there is exactly one $i$ from $1$ to $n$ such that $p_i = x$.

Input

Each test contains multiple test cases. The first line contains the number of test cases $t$ ($1 \le t \le 50,000$). The description of the test cases follows.

The first line of each test case contains a single integer $n$ ($1 \le n \le 300,000$) — the length of the array $a$.

The second line of each test case contains $n$ integers $a_1, a_2, \ldots, a_n$ ($-10^9 \le a_i \le 10^9$) — elements of the array $a$. It is guaranteed that the sum of the array $a$ is zero, in other words: $a_1 + a_2 + \ldots + a_n = 0$.

It is guaranteed that the sum of $n$ over all test cases does not exceed $300,000$.

Output

For each test case, if it is impossible to rearrange the elements of the array $a$ in the required way, print “No” in a single line.

If possible, print “Yes” in the first line, and then in a separate line $n$ numbers — elements $a_1, a_2, \ldots, a_n$ rearranged in a valid order ($a_{p_1}, a_{p_2}, \ldots, a_{p_n}$).

If there are several possible answers, you can output any of them.

Solution

这是一道构造的题目，注意到题目有限定条件：$\sum a_i=0$。

首先考虑有正有负的情况。此时，我们从左向右构造新数组，同时维护数组的前缀和 $s$。

  • 如果 $s\ge 0$，那么就添加一个负数到末尾。
  • 如果 $s\lt 0$，那么就添加一个非负数到末尾。

因为保证了数列之和为 $0$，所以当 $s\ge 0$ 时，你总能找到还没添加的非正数，同理第二种情况你也总能找到未添加的正数。

接下来，我们证明这个策略的正确性。

【结论】：$\min a_i\le s\le \max a_i$

我们使用数学归纳法。当我们添加第一个数的时候，此时的 $s$ 就是某一个数 $a_i$，这个结论显然成立。

随后，我们添加数。因为我们添加的数 $a_k$ 和 $s$ 总是符号相反，所以

  • $s\ge 0$ 时，那么如果要使结论不成立， $a_k+s\lt \min a_i$，所以 $a_k\lt \min a_i$，这与 $\min a_i$ 的定义矛盾！
  • $s\lt 0$ 时，同理。

所以，这样的策略下，这个结论总是成立的。

我们接着再证明在这样的策略下，对于任意区间，题目所给的不等式一定成立。因为任意区间可以写成两个前缀和的差，$\sum_{i=l}^{r} a_i=S_r-S_{l-1}$，所以 $|S_r-S_{l-1}|\le\max a_i-\min a_i$（因为两个前缀和不可能同时取到 $\min$ 和 $\max$）

特殊情况是：全部为 $0$ 是无解的。这里特判一下即可。

CF1572A - Book

发表于 2024-02-28 更新于 2024-03-18 分类于 ACM ， Codeforces ， 1572

问题描述

You are given a book with $n$ chapters.

Each chapter has a specified list of other chapters that need to be understood in order to understand this chapter. To understand a chapter, you must read it after you understand every chapter on its required list.

Currently you don’t understand any of the chapters. You are going to read the book from the beginning till the end repeatedly until you understand the whole book. Note that if you read a chapter at a moment when you don’t understand some of the required chapters, you don’t understand this chapter.

Determine how many times you will read the book to understand every chapter, or determine that you will never understand every chapter no matter how many times you read the book.

Input

Each test contains multiple test cases. The first line contains the number of test cases $t$ ($1 \le t \le 2\cdot10^4$).

The first line of each test case contains a single integer $n$ ($1 \le n \le 2\cdot10^5$) — number of chapters.

Then $n$ lines follow. The $i$-th line begins with an integer $k_i$ ($0 \le k_i \le n-1$) — number of chapters required to understand the $i$-th chapter. Then $k_i$ integers $a_{i,1}, a_{i,2}, \dots, a_{i, k_i}$ ($1 \le a_{i, j} \le n, a_{i, j} \ne i, a_{i, j} \ne a_{i, l}$ for $j \ne l$) follow — the chapters required to understand the $i$-th chapter.

It is guaranteed that the sum of $n$ and sum of $k_i$ over all testcases do not exceed $2\cdot10^5$.

Output

For each test case, if the entire book can be understood, print how many times you will read it, otherwise print $-1$.

Solution

我们首先把章节之间的关系抽象成一条有向边：如果 $a\to b$，那么就说明 $a$ 是 $b$ 的前置章节。只有所有前置章节被读完才能读这一章，我们可以联想到拓扑排序。

不过和一般的拓扑排序不同的是，我们每次读，只能顺序的读，如果先读了 $u$，那么想要再次读 $u-1$ 只能放到下一轮（也就是总的阅读次数 $+1$）。所以我们可以把阅读的前后顺序也抽象一下，抽象成有向边的边权，如果 $w{a\to b}=1$，就说明 $a$ 虽然是 $b$ 的前置，但是由于我们先阅读 $a$，然后才能去阅读 $b$，那么阅读 $b$ 只能放到下一轮，也就是总的阅读次数 $+1$；如果 $w{a\to b}=0$，就说明可以同一个轮次里先阅读 $a$ 再阅读 $b$。

求解次数就可以转换成一个 $\text{DAG}$ 上的 $\text{DP}$ 问题，这里，$\text{DP}$ 策略的选择应该是设置成 $dp_i=0$ 然后取 $\max$ 更新，而不是设置成 $\text{INF}$ 然后取 $\min$ 更新。原因在于，考虑 $a$ 有前置 $b,c$，并且有 $b<a,c>a$，那么如果取 $\min$ 更新，那么 $dp_a=\min(dp_b+w{b\to a},dp_c+w{c\to a})$，然而这个 $\text{DP}$ 表达式的实际含义是只要 $b,c$ 其中的一个被阅读了，我就可以读 $a$，这与题意是不符的；取 $\max$ 的实际含义才是只有全部读完前置才能读 $a$。

当然，有可能有无解的情况。这种情况下，就说明 $\text{DAG}$ 上有环。我们可以统计拓扑处理过的点的个数 cnt，如果 cnt 不等于总的点数，就说明有环，永远不可能读完前置。

CF1933E - Optimal Training

发表于 2024-02-27 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1933

原题链接

问题描述

Isaac 开始他的训练。有 $n$ 条跑道可用，第 $i$ 条跑道 （$1 \le i \le n$）由 $a_i$ 个等长的小节组成。

给定整数 $u$（$1 \le u \le 10^9$），完成每个小节可以增加 Isaac 的能力一定值，描述如下：

  • 完成第 $1$ 个小节会使 Isaac 的表现提高 $u$。
  • 完成第 $2$ 个小节会使 Isaac 的表现提高 $u-1$。
  • 完成第 $3$ 个小节会使 Isaac 的表现提高 $u-2$。
  • $\ldots$
  • 完成第 $k$ 个小节（$k \ge 1$）会使 Isaac 的表现提高 $u+1-k$。（值 $u+1-k$ 可以是负数，这意味着完成额外的小节会降低 Isaac 的表现。）

你还会得到一个整数 $l$。你必须选择一个整数 $r$，使得 $l \le r \le n$，Isaac 将完成每个跑道的每个小节 $l, l + 1, \dots, r$（即，总共 $\sum_{i=l}^r a_i = a_l + a_{l+1} + \ldots + a_r$ 个小节）。

回答以下问题：你可以选择的最佳 $r$ 是多少，使得 Isaac 的表现提升最大？

如果有多个 $r$ 可以使 Isaac 的表现提升最大，输出最小的 $r$。

为增加难度，你需要回答 $q$ 个不同的 $l$ 和 $u$ 值的问题。

Input

The first line of input contains a single integer $t$ ($1 \le t \le 10^4$) — the number of test cases.

The descriptions of the test cases follow.

The first line contains a single integer $n$ ($1 \le n \le 10^5$).

The second line contains $n$ integers $a_1, a_2, \ldots, a_n$ ($1 \le a_i \le 10^4$).

The third line contains a single integer $q$ ($1 \le q \le 10^5$).

The next $q$ lines each contain two integers $l$ and $u$ ($1 \le l \le n, 1 \le u \le 10^9$) — the descriptions to each query.

The sum of $n$ over all test cases does not exceed $2 \cdot 10^5$. The sum of $q$ over all test cases does not exceed $2 \cdot 10^5$.

Output

For each test case, output $q$ integers: the $i$-th integer contains the optimal $r$ for the $i$-th query. If there are multiple solutions, output the smallest one.

Solution

我们翻译一下题目的意思，其实就是在说：对于一个给定的 $a$ 数列、一个区间的左端点 $l$ 和一个 $u$，希望 $\sum_{i=l}^{r} a_i$ 尽可能接近 $u$。我们先想一下，如果区间的总和是 $k$，那么这一段区间能获得的能力值是 $S=u+\ldots+(u+1-k)=\frac{k\times(2u+1-k)}{2}$。我们希望 $S$ 尽可能接近 $u$，因为如果等于 $u+1$ 的话，$u+1-(u+1)=0$，他对能力值提升的贡献是 $0$，就可以扔掉这一段。

所以，我们需要快速处理出区间的和这一信息，这一部分可以使用前缀和实现。

接着，我们需要在前缀和上进行二分来查找总和最接近 $u$ 的区间。我们可以将 pre[x] - pre[l-1] > u 作为二分的分界线。但是，我们仍需要对这个分界点做一下讨论：因为可能分界点左右的区间和是 $u-1$ 和 $u+5$，而前者区间能获得的能力值大于后者，所以，需要额外判断一下。

CF1219C - Periodic Integer Number

发表于 2024-02-27 更新于 2024-03-14 分类于 Codeforces ， 1219

CF1219C - 周期整数

Problem Description

Alice became interested in periods of integer numbers. We say positive $X$ integer number is periodic with length $L$ if there exists positive integer number $P$ with $L$ digits such that $X$ can be written as $PPPP…P$. For example:

$X = 123123123$ is periodic number with length $L = 3$ and $L = 9$

$X = 42424242$ is periodic number with length $L = 2,L = 4$ and $L = 8$

$X = 12345$ is periodic number with length $L = 5$

For given positive period length $L$ and positive integer number $A$, Alice wants to find smallest integer number $X$ strictly greater than $A$ that is periodic with length $L$.

Input

First line contains one positive integer number $L \ (1 \leq L \leq 10^5)$ representing length of the period. Second line contains one positive integer number $A \ (1 \leq A \leq 10^{100 000})$.

Output

One positive integer number representing smallest positive number that is periodic with length $L$ and is greater than $A$.

Solution

本质是对 $L$ 与 $N$ 的大小关系进行分类讨论。注重逻辑，较为繁复。

  1. 如果 $L>N$，则说明 $A$ 的位数不够，此时只需要构造 $100\ldots0$ 即可。
  2. 如果 $N % L\ne0$，此时最小的合法的数的位数一定超过 $A$ 的位数，可以构造 $\begin{matrix} \underbrace{\begin{matrix} \underbrace{100\ldots 0} \ L位 \end{matrix}\begin{matrix} \underbrace{100\ldots 0} \ L位 \end{matrix} \cdots \begin{matrix} \underbrace{100\ldots 0} \ L位 \end{matrix}} \ N/L个 \end{matrix}$ 。
  3. 否则 $N%L=0$，因为要生成的數最小，贪心地考虑，尝试先取前 $L$ 个数字组成 $X$，并将 $X$ 重复。此时仍有三种情况：
    1. 如果不是 $X=99\ldots9$ 的形式
      1. 先看 $X$ 重复多次的情况
      2. 那么可以令 $X’ = X+1$，这样新数在第 $L$ 位大于 $A$，且尽可能的小。
    2. 如果 $X=99\ldots9$ 的形式
      1. 如果重复多次发现大于 $A$，则可以成为解。例如 $L=3,\ A=999998$，此时 $X=999$，生成的数为 $999\ 999\gt999\ 998$。
      2. 如果不大于，例如 $A=999999,\ L=3$，此时 $X+1$ 也超出了 $L$ 位。解决办法是，重复次数 $+1$，然后要求最小， $X=\begin{matrix} \underbrace{100\ldots 0} \ L \end{matrix}$。

CF161D - Distance in Tree

发表于 2024-02-26 更新于 2024-03-14 分类于 ACM ， Codeforces ， 161

Problem Description

一棵树是一个不包含任何环的连通图。

树的两个顶点之间的距离是这些顶点之间最短路径的长度（以边为单位）。

给定一棵有 $n$ 个顶点的树和一个正数 $k$ 。找到具有距离正好为 $k$ 的顶点对的不同对数。请注意，对 $(v, u)$ 和 $(u, v)$ 被视为相同的一对。

输入

  • $n\le 50000, k\le 500$
  • 接下来 $n-1$ 行，每行两个正整数 $a_i,b_i$，表示 $a_i,b_i$ 之间有一条边。

输出

输出一个整数，表示树上有多少点对，满足他们的距离为 $k$。

Solution

这是一道点分治的模板题。点分治适合处理大规模树上路径问题。

我们先选定树上的一个节点 $u$ 作为根，那么我们可以吧树上的所有路径划分成两类：

  1. 经过 $u$ 的路径
  2. 不经过 $u$ 的路径

而由于父亲-儿子的关系，我们可以使用递归处理第二类路径。我们所要做的，就是考虑如何处理第一类路径。

考虑两个点 $u,v$，他们之间的路径经过根节点 $rt$，那么我们可以将这条路径拆成两部分看：一部分是 $u\to rt$，另一部分是 $rt\to v$。如果 $u\to v$ 是一条合法的路径，那么也就是说 $d_{u\to rt}+d_{v\to rt}=K$。

如果给定一个根节点，我们可以通过 $\text{DFS}$ 预先处理好 $dis$ 数组（$dis[i]$ 表示到根节点距离为 $i$ 的节点个数），这样就方便了后续的计算。假定当前树的根节点是 $x$，那么我们枚举 $i\in [0,K]$，$i$ 表示路径的一端到 $x$ 距离为 $i$，那么另一端的距离就是 $K-i$，把 $dis[i]$ 和 $dis[K-i]$ 相乘即可。

代码中 dis[d[u]]++ 是统计节点数的，统计距离根节点为 $i$ 的点数有几个。并且加入了一个剪枝优化：如果当前的 d[u] 已经超过 $K$，那么 $u$ 的子节点到根节点的距离也一定大于 $K$，不会对答案产生贡献。

计算以 $u$ 为根节点时有多少方案数。要注意的是，如果 $K$ 是偶数，那么就会在 $dis[\frac{K}{2}]$ 中选两个点，方案数并不是直接自己乘自己。

这样存在一个小bug，如果同时被算入 $dis[i]$ 和 $dis[K-i]$ 的两个点在同一棵子树上，这样的路径其实是不合法的，因为路径没有经过根节点。所有要减去这一部分的方案。但是我们发现他们到根节点的路径都经过根节点的某个子节点，这个子节点到根节点的距离都是 $1$。所以我们可以用 calc(v, 1, -1) 算出不经过根节点的方案数减一下即可。

但是这样会有一个问题：如果树的形态是一条链，那么如果根节点选到了链的一端，每次递归下去就相当于只递归到链上的下一个点，整体的时间复杂度会达到 $O(N)$。所以，我们在预处理之前，需要先求出树的重心，然后将树的中心作为根节点进行递归。

注意此处的 if 判断中是 sum/2 （sum表示子树的大小），是因为我们要找的是子树的根节点，而不是整棵树的根节点。

ABC340G - Leaf Colors

发表于 2024-02-25 更新于 2024-03-14 分类于 ACM ， AtCoder Beginner Round ， 340

Problem Description

ABC340G - Leaf Colors

题面翻译

给出一棵节点数为 $N$ 的树 $T=(V,E)$，节点有颜色，节点 $i$ 的颜色为 $A_i$。

求满足以下条件的 $V$ 的子集 $S$ 的数量：

  • $S$ 的导出子图 $G$ 是一棵树；
  • $G$ 的叶节点颜色一致。

答案对 $998244353$ 取模。

导出子图：点集 $S$ 的导出子图 $G$ 的点集为 $S$，边集为 ${(u,v)|u,v\in S\land (u,v)\in E}$。

输入格式

入力は以下の形式で標準入力から与えられる。

$N$
$A_1$ $A_2$ $\dots$ $A_N$
$u_1$ $v_1$
$u_2$ $v_2$
$\vdots$
$u_{N-1}$ $v_{N-1}$

输出格式

問題文の条件を満たす頂点集合の(非空な)部分集合 $S$ の個数を $998244353$ で割った余りを出力せよ。

制約

  • $1\ \leq\ N\ \leq\ 2\ \times\ 10^5$
  • $1\ \leq\ A_i\ \leq\ N$
  • $1\ \leq\ u_i\ \lt\ v_i\ \leq\ N$
  • 入力で与えられるグラフは木
  • 入力される値は全て整数

Solution

我们首先要考虑到计数DP。先简化问题，由于对于一棵树而言，只有根节点、叶子节点才有可能度数为 $1$，如果让叶子节点的颜色固定，那么个数是比较好求的。

我们的大致思路是：枚举颜色 $C$，然后求叶子节点颜色都是 $C$ 的树的个数。

然后我们可以发现，假定枚举的颜色是 $C$，如果某节点 $V_0$ 的颜色不是 $C$，那么 $V_0$ 不可能作为叶子节点，所以，我们可以建立虚树，只在颜色为 $C$ 的节点上建树。为了保证虚树是连通的，我们添加他们之间的 $LCA$.

然后我们枚举颜色 $C$。考虑 $dp[u]$ 表示：选择了 $u$ 时，有多少树是以 $u$ 为根、且所有叶子节点的颜色都是 $C$. 所以， $dp[u]=\prod_{v\in son[u]}(dp[v]+1)$. 意义是，只要叶子节点相同，可以看成是子树 $v_1$ 与子树 $v_2$ 通过 $u$ 连在一起，形成一张新的图，并且这张图是一棵树； $+1$ 是因为还可以不选择子树 $v$ 内的任何节点，单独一个 $u$ 也是合法的答案。

但是，还要考虑到 $u$ 的颜色不是 $C$ 的情况。如果 $u$ 的颜色不是 $C$，那么如果只选择一棵 $u$ 的子树和 $u$ 组成新的图，在这张新的图中， $u$ 的度数也是 $1$，但 $u$ 的颜色不是 $C$，这个答案是不合法的；此外，如果都不选，单独 $u$ 本身也不能成为合法的解。所以，总共有 $(\sum dp[v])+1$ 种不合法的解。
