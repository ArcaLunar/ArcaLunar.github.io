CF1941G - Subway

发表于 2024-03-11 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1941

题目

题解 （01-BFS）

因为题目要求 a→ba\to ba→b 的路径上最少要经过几条地铁路线，因为“最少”这个关键词，我们会想到最短路。

我们来思考一下地铁线路最少的路径上的颜色分布。如果经过的边存在“红 →\to→ 黄 →\to→ 红”这样的情况，这实际上并不是最优的，因为题目中说“相同颜色的边都是联通的”，所以我们完全可以不走黄色的边，只走红色边，使答案 −1-1−1。

最短路肯定是需要边权的，所以，我们接下来考虑如何将颜色信息建模为边权信息。对于一个点 uuu 来说，比如说它从颜色为 c1c_1c1​ 的边走过来，那么我们遍历和它相连的边，如果也是 c1c_1c1​ 那么答案是 +0+0+0 的，如果不是 c1c_1c1​ 那么答案是 +1+1+1 的。

法一：虚拟点，跑 Dijkstra

一种办法，我们建立虚拟点表示边的颜色信息。我们把边也看成特殊的点，a→colorba\xrightarrow{color} bacolor​b 建模成 a→0color→1ba\xrightarrow{0}color\xrightarrow{1}ba0​color1​b。那我们考虑同一条地铁线路 a→colorb→colorc→colorda\xrightarrow{color} b\xrightarrow{color} c\xrightarrow{color} dacolor​bcolor​ccolor​d，其实就等效于 a→0color→1da\xrightarrow{0}color\xrightarrow{1}da0​color1​d。

法二：01-BFS

01-BFS 的本质其实就是 Dijkstra，我们用 deque 维护 dis[]，如果边权为 000 放入队首，否则放入队尾。实际上，我们是在用 deque 模拟 priority_queue（deque 内维护的 dis[] 仍然满足单调性）。

我们 deque 里存放三个东西：

  1. 最短路距离

洛谷P2146 - [NOI2015] 软件包管理器

发表于 2024-03-11 更新于 2024-03-14 分类于 ACM ， 洛谷

题目

题目背景

Linux 用户和 OSX 用户一定对软件包管理器不会陌生。通过软件包管理器，你可以通过一行命令安装某一个软件包，然后软件包管理器会帮助你从软件源下载软件包，同时自动解决所有的依赖（即下载安装这个软件包的安装所依赖的其它软件包），完成所有的配置。Debian/Ubuntu 使用的 apt-get，Fedora/CentOS 使用的 yum，以及 OSX 下可用的 homebrew 都是优秀的软件包管理器。

题目描述

你决定设计你自己的软件包管理器。不可避免地，你要解决软件包之间的依赖问题。如果软件包 aaa 依赖软件包 bbb，那么安装软件包 aaa 以前，必须先安装软件包 bbb。同时，如果想要卸载软件包 bbb，则必须卸载软件包 aaa。

现在你已经获得了所有的软件包之间的依赖关系。而且，由于你之前的工作，除 000 号软件包以外，在你的管理器当中的软件包都会依赖一个且仅一个软件包，而 000 号软件包不依赖任何一个软件包。且依赖关系不存在环（即不会存在 mmm 个软件包 a1,a2,…,ama_1,a_2, \dots , a_ma1​,a2​,…,am​，对于 i<mi<mi<m，aia_iai​ 依赖 ai+1a_{i+1}ai+1​，而 ama_mam​ 依赖 a1a_1a1​ 的情况）。

现在你要为你的软件包管理器写一个依赖解决程序。根据反馈，用户希望在安装和卸载某个软件包时，快速地知道这个操作实际上会改变多少个软件包的安装状态（即安装操作会安装多少个未安装的软件包，或卸载操作会卸载多少个已安装的软件包），你的任务就是实现这个部分。

注意，安装一个已安装的软件包，或卸载一个未安装的软件包，都不会改变任何软件包的安装状态，即在此情况下，改变安装状态的软件包数为 000。

输入格式

第一行一个正整数 nnn，表示软件包个数，从 000 开始编号。
第二行有 n−1n-1n−1 个整数，第 iii 个表示 iii 号软件包依赖的软件包编号。
然后一行一个正整数 qqq，表示操作个数，格式如下：

  • install x 表示安装 xxx 号软件包
  • uninstall x 表示卸载 xxx 号软件包

一开始所有软件包都是未安装的。

对于每个操作，你需要输出这步操作会改变多少个软件包的安装状态，随后应用这个操作（即改变你维护的安装状态）。

输出格式

输出 qqq 行，每行一个整数，表示每次询问的答案。

题解

我们比较容易想到的是，安装一个软件需要所有前置依赖，而每个软件只有一个前置依赖，所以整个软件包管理系统构成了一棵有根树，安装一个软件 ppp 在这棵依赖树上对应的操作是把 p→0p\to 0p→0 的路径上所有软件设置成 111（这里假定 111 表示已安装，000 表示未安装）。而删除一个软件对应的是，把以 ppp 为根节点的子树都设置成 000。所以，我们可以结合树链剖分和区修区查线段树（要注意的是，由于软件下标从 000 开始，所以代表懒标记的 tag[] 的默认值应该设置成 −1-1−1，而不是 000）来完成这些操作。

CF1872G - Replace with Product

发表于 2024-03-07 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1872

题目描述

Given an array aaa of nnn positive integers. You need to perform the following operation exactly once:

  • Choose 222 integers lll and rrr (1≤l≤r≤n1 \le l \le r \le n1≤l≤r≤n) and replace the subarray a[l…r]a[l \ldots r]a[l…r] with the single element: the product of all elements in the subarray (al⋅…⋅ar)(a_l \cdot \ldots \cdot a_r)(al​⋅…⋅ar​).

Your task is to maximize the sum of the array after applying this operation. Find the optimal subarray to apply this operation.

Input

Each test consists of multiple test cases. The first line contains a single integer ttt (1≤t≤1041 \le t \le 10^41≤t≤104) — the number of test cases. This is followed by the description of the test cases.

The first line of each test case contains a single number nnn (1≤n≤2⋅1051 \le n \le 2 \cdot 10^51≤n≤2⋅105) — the length of the array aaa.

题解

我们先想一个简单情况，只有两个数 a,ba,ba,b，那么只要 ab>a+bab>a+bab>a+b 我就要把 a,ba,ba,b 用乘积替换，而且，感性地来说，如果固定 bbb，那么我的 aaa 越大，ababab 相比 a+ba+ba+b 就越大，对于答案的贡献也越大，所以更应该把 aaa 乘进子数组里。总结一下，aaa 越大，对答案的作用也越大。

换而言之，aaa “作用最小”的时候是 a=2a=2a=2，在最坏情况下，max⁡ai=2\max a_i=2maxai​=2，所以，我们需要判断当用乘积替换的时候，是不是一定能够保证大于其和。我们可以猜测，只要当 ∏ai>2N\prod a_i\gt 2N∏ai​>2N 时，就一定满足 ∏ai≥∑ai\prod a_i\ge \sum a_i∏ai​≥∑ai​。

当 ∏ai≤2N\prod a_i\le 2N∏ai​≤2N 时，我们就可以暴力枚举了，因为此时数的个数很少，O(N2)O(N^2)O(N2) 足够枚举。

题面描述

The girls of SYSU like drinking tea. But one day, they wanted a change and decided to try coffee in the next nnn days.

Notice that each coupon can be used only once and Zayin can use more than one coupon per day. The price can be a negative number after using the coupons.

Since the girls of SYSU still need to drink tea, Zayin decided to choose some days to buy coffee. Now she wants to know the minimum money she will spend (or the maximum money she can get) if she chooses exactly k (1≤k≤n)k\ (1\le k \le n)k (1≤k≤n) days to buy coffee.

Input

The first line of the input contains a single integer t (1≤t≤10)t\ (1\le t\le 10)t (1≤t≤10) — the number of test cases. The description of the test cases follows.

The first line contains two integers n,m (1≤n,m≤2×105)n,m\ (1\le n, m\le 2\times 10^5)n,m (1≤n,m≤2×105) - the number of days and the number of coupons.

The second line contains nnn integers a1,a2,⋯ ,an (1≤ai≤109)a_1,a_2,\cdots,a_n\ (1\le a_i \le 10^9)a1​,a2​,⋯,an​ (1≤ai​≤109) - aia_iai​ denotes the price of coffee on day iii.

Next mmm lines, each line contains two integers ri,wi (1≤ri≤n, 1≤wi≤109)r_i,w_i\ (1\le r_i\le n,\ 1\le w_i\le 10^9)ri​,wi​ (1≤ri​≤n, 1≤wi​≤109), denoting that the iii th coupon can be used before day rir_iri​ and can reduce the price by wiw_iwi​.

Output

For each testcase, output nnn integers - the iii th integer represents the minimum money Zayin will spend to buy coffee if she chooses exactly iii days to buy coffee.

Notice that the answer can be a negative integer.

题解

我们从 k=1k=1k=1 时入手。如果只用一天买咖啡，那么贪心地考虑的话，一定是希望大量优惠券用在同一天买咖啡，这样打折的力度更大，那么我们就希望价格减去优惠的最终价格最小。所以，对于 k=1k=1k=1 时，我们的策略就是，对于第 iii 张优惠券，我们在 1∼ri1\sim r_i1∼ri​ 中找一个价格最低的时候，然后把优惠券用在这一天（记为 mim_imi​）。

当 k>1k>1k>1 时，优惠券用在其他时间（t∈[1,ri],t≠mit\in [1,r_i], t\ne m_it∈[1,ri​],t=mi​）和用在 mim_imi​ 是一样的，因为只要用了优惠券，那么价格就一定减少 wiw_iwi​，这和优惠券在什么时候使用是无关的。

所以我们可以对优惠券按过期时间 rir_iri​ 进行排序，从小到大维护价格的最低点，在这一点减去优惠力度，最后对最终价格排序并计算前缀和即可。

Solution

这是一道构造的题目，注意到题目有限定条件：∑ai=0\sum a_i=0∑ai​=0。

首先考虑有正有负的情况。此时，我们从左向右构造新数组，同时维护数组的前缀和 sss。

  • 如果 s≥0s\ge 0s≥0，那么就添加一个负数到末尾。
  • 如果 s<0s\lt 0s<0，那么就添加一个非负数到末尾。

因为保证了数列之和为 000，所以当 s≥0s\ge 0s≥0 时，你总能找到还没添加的非正数，同理第二种情况你也总能找到未添加的正数。

接下来，我们证明这个策略的正确性。

【结论】：min⁡ai≤s≤max⁡ai\min a_i\le s\le \max a_iminai​≤s≤maxai​

我们使用数学归纳法。当我们添加第一个数的时候，此时的 sss 就是某一个数 aia_iai​，这个结论显然成立。

随后，我们添加数。因为我们添加的数 aka_kak​ 和 sss 总是符号相反，所以

  • s≥0s\ge 0s≥0 时，那么如果要使结论不成立， ak+s<min⁡aia_k+s\lt \min a_iak​+s<minai​，所以 ak<min⁡aia_k\lt \min a_iak​<minai​，这与 min⁡ai\min a_iminai​ 的定义矛盾！
  • s<0s\lt 0s<0 时，同理。

所以，这样的策略下，这个结论总是成立的。

我们接着再证明在这样的策略下，对于任意区间，题目所给的不等式一定成立。因为任意区间可以写成两个前缀和的差，∑i=lrai=Sr−Sl−1\sum_{i=l}^{r} a_i=S_r-S_{l-1}∑i=lr​ai​=Sr​−Sl−1​，所以 ∣Sr−Sl−1∣≤max⁡ai−min⁡ai|S_r-S_{l-1}|\le\max a_i-\min a_i∣Sr​−Sl−1​∣≤maxai​−minai​（因为两个前缀和不可能同时取到 min⁡\minmin 和 max⁡\maxmax）
