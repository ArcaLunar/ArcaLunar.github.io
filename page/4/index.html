CF161D - Distance in Tree

发表于 2024-02-26 更新于 2024-03-14 分类于 ACM ， Codeforces ， 161

Problem Description

一棵树是一个不包含任何环的连通图。

树的两个顶点之间的距离是这些顶点之间最短路径的长度（以边为单位）。

给定一棵有 nnn 个顶点的树和一个正数 kkk 。找到具有距离正好为 kkk 的顶点对的不同对数。请注意，对 (v,u)(v, u)(v,u) 和 (u,v)(u, v)(u,v) 被视为相同的一对。

输入

  • n≤50000,k≤500n\le 50000, k\le 500n≤50000,k≤500
  • 接下来 n−1n-1n−1 行，每行两个正整数 ai,bia_i,b_iai​,bi​，表示 ai,bia_i,b_iai​,bi​ 之间有一条边。

输出

输出一个整数，表示树上有多少点对，满足他们的距离为 kkk。

Solution

这是一道点分治的模板题。点分治适合处理大规模树上路径问题。

我们先选定树上的一个节点 uuu 作为根，那么我们可以吧树上的所有路径划分成两类：

  1. 经过 uuu 的路径
  2. 不经过 uuu 的路径

而由于父亲-儿子的关系，我们可以使用递归处理第二类路径。我们所要做的，就是考虑如何处理第一类路径。

考虑两个点 u,vu,vu,v，他们之间的路径经过根节点 rtrtrt，那么我们可以将这条路径拆成两部分看：一部分是 u→rtu\to rtu→rt，另一部分是 rt→vrt\to vrt→v。如果 u→vu\to vu→v 是一条合法的路径，那么也就是说 du→rt+dv→rt=Kd_{u\to rt}+d_{v\to rt}=Kdu→rt​+dv→rt​=K。

如果给定一个根节点，我们可以通过 DFS\text{DFS}DFS 预先处理好 disdisdis 数组（dis[i]dis[i]dis[i] 表示到根节点距离为 iii 的节点个数），这样就方便了后续的计算。假定当前树的根节点是 xxx，那么我们枚举 i∈[0,K]i\in [0,K]i∈[0,K]，iii 表示路径的一端到 xxx 距离为 iii，那么另一端的距离就是 K−iK-iK−i，把 dis[i]dis[i]dis[i] 和 dis[K−i]dis[K-i]dis[K−i] 相乘即可。

代码中 dis[d[u]]++ 是统计节点数的，统计距离根节点为 iii 的点数有几个。并且加入了一个剪枝优化：如果当前的 d[u] 已经超过 KKK，那么 uuu 的子节点到根节点的距离也一定大于 KKK，不会对答案产生贡献。

计算以 uuu 为根节点时有多少方案数。要注意的是，如果 KKK 是偶数，那么就会在 dis[K2]dis[\frac{K}{2}]dis[2K​] 中选两个点，方案数并不是直接自己乘自己。

这样存在一个小bug，如果同时被算入 dis[i]dis[i]dis[i] 和 dis[K−i]dis[K-i]dis[K−i] 的两个点在同一棵子树上，这样的路径其实是不合法的，因为路径没有经过根节点。所有要减去这一部分的方案。但是我们发现他们到根节点的路径都经过根节点的某个子节点，这个子节点到根节点的距离都是 111。所以我们可以用 calc(v, 1, -1) 算出不经过根节点的方案数减一下即可。

但是这样会有一个问题：如果树的形态是一条链，那么如果根节点选到了链的一端，每次递归下去就相当于只递归到链上的下一个点，整体的时间复杂度会达到 O(N)O(N)O(N)。所以，我们在预处理之前，需要先求出树的重心，然后将树的中心作为根节点进行递归。

注意此处的 if 判断中是 sum/2 （sum表示子树的大小），是因为我们要找的是子树的根节点，而不是整棵树的根节点。

ABC340G - Leaf Colors

发表于 2024-02-25 更新于 2024-03-14 分类于 ACM ， AtCoder Beginner Round ， 340

Problem Description

ABC340G - Leaf Colors

题面翻译

给出一棵节点数为 NNN 的树 T=(V,E)T=(V,E)T=(V,E)，节点有颜色，节点 iii 的颜色为 AiA_iAi​。

求满足以下条件的 VVV 的子集 SSS 的数量：

  • SSS 的导出子图 GGG 是一棵树；
  • GGG 的叶节点颜色一致。

答案对 998244353998244353998244353 取模。

导出子图：点集 SSS 的导出子图 GGG 的点集为 SSS，边集为 {(u,v)∣u,v∈S∧(u,v)∈E}\{(u,v)|u,v\in S\land (u,v)\in E\}{(u,v)∣u,v∈S∧(u,v)∈E}。

输入格式

入力は以下の形式で標準入力から与えられる。

NNN
A1A_1A1​ A2A_2A2​ …\dots… ANA_NAN​
u1u_1u1​ v1v_1v1​
u2u_2u2​ v2v_2v2​
⋮\vdots⋮
uN−1u_{N-1}uN−1​ vN−1v_{N-1}vN−1​

输出格式

問題文の条件を満たす頂点集合の(非空な)部分集合 SSS の個数を 998244353998244353998244353 で割った余りを出力せよ。

制約

  • 1 ≤ N ≤ 2 × 1051\ \leq\ N\ \leq\ 2\ \times\ 10^51 ≤ N ≤ 2 × 105
  • 1 ≤ Ai ≤ N1\ \leq\ A_i\ \leq\ N1 ≤ Ai​ ≤ N
  • 1 ≤ ui < vi ≤ N1\ \leq\ u_i\ \lt\ v_i\ \leq\ N1 ≤ ui​ < vi​ ≤ N
  • 入力で与えられるグラフは木
  • 入力される値は全て整数

Solution

我们首先要考虑到计数DP。先简化问题，由于对于一棵树而言，只有根节点、叶子节点才有可能度数为 111，如果让叶子节点的颜色固定，那么个数是比较好求的。

我们的大致思路是：枚举颜色 CCC，然后求叶子节点颜色都是 CCC 的树的个数。

然后我们可以发现，假定枚举的颜色是 CCC，如果某节点 V0V_0V0​ 的颜色不是 CCC，那么 V0V_0V0​ 不可能作为叶子节点，所以，我们可以建立虚树，只在颜色为 CCC 的节点上建树。为了保证虚树是连通的，我们添加他们之间的 LCALCALCA.

然后我们枚举颜色 CCC。考虑 dp[u]dp[u]dp[u] 表示：选择了 uuu 时，有多少树是以 uuu 为根、且所有叶子节点的颜色都是 CCC. 所以， dp[u]=∏v∈son[u](dp[v]+1)dp[u]=\prod_{v\in son[u]}(dp[v]+1)dp[u]=∏v∈son[u]​(dp[v]+1). 意义是，只要叶子节点相同，可以看成是子树 v1v_1v1​ 与子树 v2v_2v2​ 通过 uuu 连在一起，形成一张新的图，并且这张图是一棵树； +1+1+1 是因为还可以不选择子树 vvv 内的任何节点，单独一个 uuu 也是合法的答案。

但是，还要考虑到 uuu 的颜色不是 CCC 的情况。如果 uuu 的颜色不是 CCC，那么如果只选择一棵 uuu 的子树和 uuu 组成新的图，在这张新的图中， uuu 的度数也是 111，但 uuu 的颜色不是 CCC，这个答案是不合法的；此外，如果都不选，单独 uuu 本身也不能成为合法的解。所以，总共有 (∑dp[v])+1(\sum dp[v])+1(∑dp[v])+1 种不合法的解。

此题中，要求递增路径，我们可以想到将无向边拆开看成 u→v,v→uu→v,v→uu→v,v→u 的两条有向边，然后只保留点权更小的点连向点权更大的点的有向边（如果 Au<AvA_u<A_vAu​<Av​，那么保留 e(u→v)e(u→v)e(u→v)）
