CF1935C - Messenger in MAC

发表于 2024-03-06 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1935

题面描述

In the new messenger for the students of the Master’s Assistance Center, Keftemerum, an update is planned, in which developers want to optimize the set of messages shown to the user. There are a total of nnn messages. Each message is characterized by two integers aia_iai​ and bib_ibi​. The time spent reading the set of messages with numbers p1,p2,…,pkp_1, p_2, \ldots, p_kp1​,p2​,…,pk​ (1≤pi≤n1 \le p_i \le n1≤pi​≤n, all pip_ipi​ are distinct) is calculated by the formula:

∑i=1kapi+∑i=1k−1∣bpi−bpi+1∣\Large \sum_{i=1}^{k} a_{p_i} + \sum_{i=1}^{k - 1} |b_{p_i} - b_{p_{i+1}}| i=1∑k​api​​+i=1∑k−1​∣bpi​​−bpi+1​​∣

Note that the time to read a set of messages consisting of one message with number p1p_1p1​ is equal to ap1a_{p_1}ap1​​. Also, the time to read an empty set of messages is considered to be 000.

The user can determine the time lll that he is willing to spend in the messenger. The messenger must inform the user of the maximum possible size of the set of messages, the reading time of which does not exceed lll. Note that the maximum size of the set of messages can be equal to 000.

The developers of the popular messenger failed to implement this function, so they asked you to solve this problem.

Input

Each test consists of multiple test cases. The first line contains a single integer ttt (1≤t≤5⋅1041 \leq t \leq 5 \cdot 10^41≤t≤5⋅104) — the number of test cases. The description of the test cases follows.

The first line of each test case contains two integers nnn and lll (1≤n≤20001 \leq n \leq 20001≤n≤2000, 1≤l≤1091 \leq l \leq 10^91≤l≤109) — the number of messages and the time the user is willing to spend in the messenger.

The iii-th of the next nnn lines contains two integers aia_iai​ and bib_ibi​ (1≤ai,bi≤1091 \le a_i, b_i \le 10^91≤ai​,bi​≤109) — characteristics of the iii-th message.

It is guaranteed that the sum of n2n^2n2 over all test cases does not exceed 4⋅1064 \cdot 10^64⋅106.

Output

For each test case, output a single integer — the maximum possible size of a set of messages, the reading time of which does not exceed lll.

题解

我们的目标，是让 S=∑api+∑∣bpi+1−bpi∣S=\sum a_{p_i}+\sum |b_{p_{i+1}}-b_{p_i}|S=∑api​​+∑∣bpi+1​​−bpi​​∣ 在不超过 LLL 的情况下，选择尽可能多的 ppp。借用贪心的思想，我们可以直观地想到，如果我的 SSS 越小，就越有可能放更多的数进来。

由于题目并没有限制 pip_ipi​ 的顺序，而 ∑∣bpi+1−bpi∣\sum |b_{p_{i+1}}-b_{p_i}|∑∣bpi+1​​−bpi​​∣ 因为带绝对值，所以我们可以认为的规定一个顺序，使得 bp1≤bp2≤bp3≤…≤bpkb_{p_1}\le b_{p_2}\le b_{p_3}\le \ldots \le b_{p_k}bp1​​≤bp2​​≤bp3​​≤…≤bpk​​。这启发我们按 bib_ibi​ 升序排序。

排完序之后呢？我们考虑贪心策略：只要总和还没超过 LLL 我就一直往里面放。如果超过了 LLL，我就先把大数扔掉，直到合法为止。之所以先扔大的，是因为如果先扔小的，那很有可能要扔掉很多个小的，才能腾出足够空间放进去数，而先大的的话，很有可能只需要扔掉一两个就可以腾出足够空间了。所以，我们在这里需要一个支持删除最大值、插入值、查询最大值的数据结构，我们可以用 priority_queue 或者 multiset 实现。

CCPC2023秦皇岛 D - Yet Another Coffee

发表于 2024-03-03 更新于 2024-03-14 分类于 ACM ， CCPC ， 2023 ， 秦皇岛

题面描述

The girls of SYSU like drinking tea. But one day, they wanted a change and decided to try coffee in the next nnn days.

Now Zayin, who always provides food and drinks for SYSU, will go to the shop to buy some coffee. She learns that the price of day iii is aia_iai​. Meanwhile, she has mmm coupons - the iii coupon can be used before day rir_iri​ (inclusively) and can reduce the price of coffee on that day by wiw_iwi​.

Notice that each coupon can be used only once and Zayin can use more than one coupon per day. The price can be a negative number after using the coupons.

Since the girls of SYSU still need to drink tea, Zayin decided to choose some days to buy coffee. Now she wants to know the minimum money she will spend (or the maximum money she can get) if she chooses exactly k (1≤k≤n)k\ (1\le k \le n)k (1≤k≤n) days to buy coffee.

Input

The first line of the input contains a single integer t (1≤t≤10)t\ (1\le t\le 10)t (1≤t≤10) — the number of test cases. The description of the test cases follows.

The first line contains two integers n,m (1≤n,m≤2×105)n,m\ (1\le n, m\le 2\times 10^5)n,m (1≤n,m≤2×105) - the number of days and the number of coupons.

The second line contains nnn integers a1,a2,⋯ ,an (1≤ai≤109)a_1,a_2,\cdots,a_n\ (1\le a_i \le 10^9)a1​,a2​,⋯,an​ (1≤ai​≤109) - aia_iai​ denotes the price of coffee on day iii.

Next mmm lines, each line contains two integers ri,wi (1≤ri≤n, 1≤wi≤109)r_i,w_i\ (1\le r_i\le n,\ 1\le w_i\le 10^9)ri​,wi​ (1≤ri​≤n, 1≤wi​≤109), denoting that the iii th coupon can be used before day rir_iri​ and can reduce the price by wiw_iwi​.

Output

For each testcase, output nnn integers - the iii th integer represents the minimum money Zayin will spend to buy coffee if she chooses exactly iii days to buy coffee.

Notice that the answer can be a negative integer.

题解

我们从 k=1k=1k=1 时入手。如果只用一天买咖啡，那么贪心地考虑的话，一定是希望大量优惠券用在同一天买咖啡，这样打折的力度更大，那么我们就希望价格减去优惠的最终价格最小。所以，对于 k=1k=1k=1 时，我们的策略就是，对于第 iii 张优惠券，我们在 1∼ri1\sim r_i1∼ri​ 中找一个价格最低的时候，然后把优惠券用在这一天（记为 mim_imi​）。

当 k>1k>1k>1 时，优惠券用在其他时间（t∈[1,ri],t≠mit\in [1,r_i], t\ne m_it∈[1,ri​],t=mi​）和用在 mim_imi​ 是一样的，因为只要用了优惠券，那么价格就一定减少 wiw_iwi​，这和优惠券在什么时候使用是无关的。

所以我们可以对优惠券按过期时间 rir_iri​ 进行排序，从小到大维护价格的最低点，在这一点减去优惠力度，最后对最终价格排序并计算前缀和即可。

CF1798D - Shocking Arrangment

发表于 2024-02-28 更新于 2024-03-14 分类于 ACM ， Codeforces ， 1798

问题描述

You are given an array a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ consisting of integers such that a1+a2+…+an=0a_1 + a_2 + \ldots + a_n = 0a1​+a2​+…+an​=0.

You have to rearrange the elements of the array aaa so that the following condition is satisfied:

\max\limits_{1 \le l \le r \le n} \lvert a_l + a_{l+1} + \ldots + a_r \rvert &lt; \max(a_1, a_2, \ldots, a_n) - \min(a_1, a_2, \ldots, a_n),

where ∣x∣|x|∣x∣ denotes the absolute value of xxx.

More formally, determine if there exists a permutation p1,p2,…,pnp_1, p_2, \ldots, p_np1​,p2​,…,pn​ that for the array ap1,ap2,…,apna_{p_1}, a_{p_2}, \ldots, a_{p_n}ap1​​,ap2​​,…,apn​​, the condition above is satisfied, and find the corresponding array.

Recall that the array p1,p2,…,pnp_1, p_2, \ldots, p_np1​,p2​,…,pn​ is called a permutation if for each integer xxx from 111 to nnn there is exactly one iii from 111 to nnn such that pi=xp_i = xpi​=x.

Input

Each test contains multiple test cases. The first line contains the number of test cases ttt (1≤t≤50 0001 \le t \le 50\,0001≤t≤50000). The description of the test cases follows.

The first line of each test case contains a single integer nnn (1≤n≤300 0001 \le n \le 300\,0001≤n≤300000) — the length of the array aaa.

The second line of each test case contains nnn integers a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ (−109≤ai≤109-10^9 \le a_i \le 10^9−109≤ai​≤109) — elements of the array aaa. It is guaranteed that the sum of the array aaa is zero, in other words: a1+a2+…+an=0a_1 + a_2 + \ldots + a_n = 0a1​+a2​+…+an​=0.

It is guaranteed that the sum of nnn over all test cases does not exceed 300 000300\,000300000.

Output

For each test case, if it is impossible to rearrange the elements of the array aaa in the required way, print “No” in a single line.

If possible, print “Yes” in the first line, and then in a separate line nnn numbers — elements a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ rearranged in a valid order (ap1,ap2,…,apna_{p_1}, a_{p_2}, \ldots, a_{p_n}ap1​​,ap2​​,…,apn​​).

If there are several possible answers, you can output any of them.

Solution

这是一道构造的题目，注意到题目有限定条件：∑ai=0\sum a_i=0∑ai​=0。

首先考虑有正有负的情况。此时，我们从左向右构造新数组，同时维护数组的前缀和 sss。

  • 如果 s≥0s\ge 0s≥0，那么就添加一个负数到末尾。
  • 如果 s<0s\lt 0s<0，那么就添加一个非负数到末尾。

因为保证了数列之和为 000，所以当 s≥0s\ge 0s≥0 时，你总能找到还没添加的非正数，同理第二种情况你也总能找到未添加的正数。

接下来，我们证明这个策略的正确性。

【结论】：min⁡ai≤s≤max⁡ai\min a_i\le s\le \max a_iminai​≤s≤maxai​

我们使用数学归纳法。当我们添加第一个数的时候，此时的 sss 就是某一个数 aia_iai​，这个结论显然成立。

随后，我们添加数。因为我们添加的数 aka_kak​ 和 sss 总是符号相反，所以

  • s≥0s\ge 0s≥0 时，那么如果要使结论不成立， ak+s<min⁡aia_k+s\lt \min a_iak​+s<minai​，所以 ak<min⁡aia_k\lt \min a_iak​<minai​，这与 min⁡ai\min a_iminai​ 的定义矛盾！
  • s<0s\lt 0s<0 时，同理。

所以，这样的策略下，这个结论总是成立的。

我们接着再证明在这样的策略下，对于任意区间，题目所给的不等式一定成立。因为任意区间可以写成两个前缀和的差，∑i=lrai=Sr−Sl−1\sum_{i=l}^{r} a_i=S_r-S_{l-1}∑i=lr​ai​=Sr​−Sl−1​，所以 ∣Sr−Sl−1∣≤max⁡ai−min⁡ai|S_r-S_{l-1}|\le\max a_i-\min a_i∣Sr​−Sl−1​∣≤maxai​−minai​（因为两个前缀和不可能同时取到 min⁡\minmin 和 max⁡\maxmax）

特殊情况是：全部为 000 是无解的。这里特判一下即可。

CF1572A - Book

发表于 2024-02-28 更新于 2024-03-18 分类于 ACM ， Codeforces ， 1572

问题描述

You are given a book with nnn chapters.

Each chapter has a specified list of other chapters that need to be understood in order to understand this chapter. To understand a chapter, you must read it after you understand every chapter on its required list.

Currently you don’t understand any of the chapters. You are going to read the book from the beginning till the end repeatedly until you understand the whole book. Note that if you read a chapter at a moment when you don’t understand some of the required chapters, you don’t understand this chapter.

Determine how many times you will read the book to understand every chapter, or determine that you will never understand every chapter no matter how many times you read the book.

Input

Each test contains multiple test cases. The first line contains the number of test cases ttt (1≤t≤2⋅1041 \le t \le 2\cdot10^41≤t≤2⋅104).

The first line of each test case contains a single integer nnn (1≤n≤2⋅1051 \le n \le 2\cdot10^51≤n≤2⋅105) — number of chapters.

Then nnn lines follow. The iii-th line begins with an integer kik_iki​ (0≤ki≤n−10 \le k_i \le n-10≤ki​≤n−1) — number of chapters required to understand the iii-th chapter. Then kik_iki​ integers ai,1,ai,2,…,ai,kia_{i,1}, a_{i,2}, \dots, a_{i, k_i}ai,1​,ai,2​,…,ai,ki​​ (1≤ai,j≤n,ai,j≠i,ai,j≠ai,l1 \le a_{i, j} \le n, a_{i, j} \ne i, a_{i, j} \ne a_{i, l}1≤ai,j​≤n,ai,j​=i,ai,j​=ai,l​ for j≠lj \ne lj=l) follow — the chapters required to understand the iii-th chapter.

It is guaranteed that the sum of nnn and sum of kik_iki​ over all testcases do not exceed 2⋅1052\cdot10^52⋅105.

Output

For each test case, if the entire book can be understood, print how many times you will read it, otherwise print −1-1−1.

Solution

我们首先把章节之间的关系抽象成一条有向边：如果 a→ba\to ba→b，那么就说明 aaa 是 bbb 的前置章节。只有所有前置章节被读完才能读这一章，我们可以联想到拓扑排序。

不过和一般的拓扑排序不同的是，我们每次读，只能顺序的读，如果先读了 uuu，那么想要再次读 u−1u-1u−1 只能放到下一轮（也就是总的阅读次数 +1+1+1）。所以我们可以把阅读的前后顺序也抽象一下，抽象成有向边的边权，如果 wa→b=1w{a\to b}=1wa→b=1，就说明 aaa 虽然是 bbb 的前置，但是由于我们先阅读 aaa，然后才能去阅读 bbb，那么阅读 bbb 只能放到下一轮，也就是总的阅读次数 +1+1+1；如果 wa→b=0w{a\to b}=0wa→b=0，就说明可以同一个轮次里先阅读 aaa 再阅读 bbb。

求解次数就可以转换成一个 DAG\text{DAG}DAG 上的 DP\text{DP}DP 问题，这里，DP\text{DP}DP 策略的选择应该是设置成 dpi=0dp_i=0dpi​=0 然后取 max⁡\maxmax 更新，而不是设置成 INF\text{INF}INF 然后取 min⁡\minmin 更新。原因在于，考虑 aaa 有前置 b,cb,cb,c，并且有 b<a,c>ab<a,c>ab<a,c>a，那么如果取 min⁡\minmin 更新，那么 dpa=min⁡(dpb+wb→a,dpc+wc→a)dp_a=\min(dp_b+w{b\to a},dp_c+w{c\to a})dpa​=min(dpb​+wb→a,dpc​+wc→a)，然而这个 DP\text{DP}DP 表达式的实际含义是只要 b,cb,cb,c 其中的一个被阅读了，我就可以读 aaa，这与题意是不符的；取 max⁡\maxmax 的实际含义才是只有全部读完前置才能读 aaa。

当然，有可能有无解的情况。这种情况下，就说明 DAG\text{DAG}DAG 上有环。我们可以统计拓扑处理过的点的个数 cnt，如果 cnt 不等于总的点数，就说明有环，永远不可能读完前置。

问题描述

Isaac 开始他的训练。有 nnn 条跑道可用，第 iii 条跑道 （1≤i≤n1 \le i \le n1≤i≤n）由 aia_iai​ 个等长的小节组成。

给定整数 uuu（1≤u≤1091 \le u \le 10^91≤u≤109），完成每个小节可以增加 Isaac 的能力一定值，描述如下：

  • 完成第 111 个小节会使 Isaac 的表现提高 uuu。
  • 完成第 222 个小节会使 Isaac 的表现提高 u−1u-1u−1。
  • 完成第 333 个小节会使 Isaac 的表现提高 u−2u-2u−2。

Problem Description

输入

  • n≤50000,k≤500n\le 50000, k\le 500n≤50000,k≤500

输出一个整数，表示树上有多少点对，满足他们的距离为 kkk。

这是一道点分治的模板题。点分治适合处理大规模树上路径问题。

我们先选定树上的一个节点 uuu 作为根，那么我们可以吧树上的所有路径划分成两类：

  1. 经过 uuu 的路径
  2. 不经过 uuu 的路径

而由于父亲-儿子的关系，我们可以使用递归处理第二类路径。我们所要做的，就是考虑如何处理第一类路径。

考虑两个点 u,vu,vu,v，他们之间的路径经过根节点 rtrtrt，那么我们可以将这条路径拆成两部分看：一部分是 u→rtu\to rtu→rt，另一部分是 rt→vrt\to vrt→v。如果 u→vu\to vu→v 是一条合法的路径，那么也就是说 du→rt+dv→rt=Kd_{u\to rt}+d_{v\to rt}=Kdu→rt​+dv→rt​=K。

代码中 dis[d[u]]++ 是统计节点数的，统计距离根节点为 iii 的点数有几个。并且加入了一个剪枝优化：如果当前的 d[u] 已经超过 KKK，那么 uuu 的子节点到根节点的距离也一定大于 KKK，不会对答案产生贡献。

计算以 uuu 为根节点时有多少方案数。要注意的是，如果 KKK 是偶数，那么就会在 dis[K2]dis[\frac{K}{2}]dis[2K​] 中选两个点，方案数并不是直接自己乘自己。

注意此处的 if 判断中是 sum/2 （sum表示子树的大小），是因为我们要找的是子树的根节点，而不是整棵树的根节点。
